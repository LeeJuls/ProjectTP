## 2. 작업 flow (단계·게이트)

```
A0 기술 스파이크 (병렬 3건)            A1 전투 완성 (S6 일부)
 ①UMG 파이프라인(S9)+클로드디자인 E2E   HP·데미지·사망·승패·스킬 소수
 ②합성 머티리얼(S2)                    (현행 씬에서, A0과 병렬)
 ③CSV 데이터 파이프라인(2.5절)

      │                                    │
      ▼                                    │

   ← 파츠 인벤토리 표 제공(작성 지원)       │
      │                                    │
A2 캐릭터 코어 (S1+S4: 파츠DB·레시피·1000풀 사전생성·골드 SaveGame)
      │                                    │
A3 드래프트 (S3: Roll/Pick/Ban 교차밴 §A-5)          │
      │                                    │
A4 대기실 (S5: 매칭 허브 Start·Opt·Quit·Gold)          │
      └──────────────┬─────────────────────┘
A5 연결: 드래프트 Final4(교차밴) → 전투 스폰 + AI 상대 (S6 나머지+S7)
      │
A6 결과 + 씬 플로우 (S8: 루프 폐쇄 — 재시작/로비분기 §A-3·부전승 §I E6·재매칭실패 §I E7·BattleResult §F·세션리셋 §A-5)

```

### 순서 논리 3가지
1. **미지수 조기 제거**: A0 스파이크 2건이 전체 구조를 좌우한다 — 합성 머티리얼이 안 되면 렌더 전략 전환(RenderTarget 굽기/소규모 풀), UMG가 안 뚫리면 UI 전략 전환. 뒤에서 터지면 재작업 폭탄.
2. **오너 가이드 대기 중 공회전 방지**: 전투 완성(A1)은 캐릭터 시스템과 독립(현행 8기 프리셋으로 진행) → A0·오너 가이드와 병렬.
3. **데이터 먼저, UI 나중**: A2(레시피·저장)가 서고 나서 A3(뽑기)·A4(대기실)가 그 위에 얹힌다. A5에서 두 갈래(캐릭터·전투)가 합류.

### 단계별 의존 메모
- A1의 HP 표시: UMG 스파이크(A0①) 결과 나오면 UMG로, 아니면 임시 월드공간 게이지 후 교체.
- **A1에 모션 배선 포함(오너 확정 2026-07-13, [[모션연결_규칙안]] E1-B안)**: 현재 `RowIndex=5` 하드코딩을 `skills.csv MotionRow` 읽기로 교체 + 행별 FrameCount 데이터화(`DT_Motions` 참조테이블 신설). 무기↔모션 연결 규칙은 B(느슨결합 — 스킬이 모션 소유, 무기는 1000풀 생성 시 성향 매칭만, 마법 룩은 완드 4색 변형).
- A2는 **오너 가이드 + A0②(합성 머티리얼)** 양쪽 뒤. 가이드가 늦으면 레시피 스키마에 태그 자리만 예약하고 진행 가능(폴백).
- A5의 "편성 파티 스폰"은 BP_BattleSpawnPoint가 레시피를 받아 외형(S2)+스탯(S1)을 구성하는 접점 — 알파 최대 통합 지점.

## 4.5 추가 논의주제 발굴 (2026-07-08, system-ui-designer+balance-designer 병렬 검토)

### 최상류 — A2(캐릭터 코어) 착수 전 필수
- **balance#1 — 전투 정체성(PositionId+GradeId)→스탯/킷 연결** [유효·긴급도 상승, 방향성2 재프레이밍]: 방향성2의 밸런싱 기준 축은 JobId가 아니라 **포지션(탱/딜/힐)** — 지급이 포지션별 2개(기획_방향성 §25)라 포지션이 롤·밴픽·구성의 실제 축이다. 따라서 balance#1을 "**PositionId + GradeId → 스탯/킷 템플릿**"으로 리드한다(직업은 포지션 하위의 킷/외형 변형 태그로 흡수 여부 검토). (구 balance#2의 "등급 변동 시 R2 동급검증 이탈·AI 등급 패리티" 우려는 배율압축 + AI도 같은 풀에서 롤로 흡수.)
