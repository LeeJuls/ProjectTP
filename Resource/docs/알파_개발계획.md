## 2. 작업 flow (단계·게이트)

```
A0 기술 스파이크 (병렬 2건)            A1 전투 완성 (S6 일부)
 ①합성 머티리얼(S2)                    HP·데미지·사망·승패·스킬 소수
 ②UMG 파이프라인(S9)                   (현행 씬에서, A0과 병렬)
      │                                    │
      ▼                                    │
 [오너 게이트] 직업/종족/성별 가이드        │
   ← 파츠 인벤토리 표 제공(작성 지원)       │
      │                                    │
A2 캐릭터 코어 (S1+S4: 파츠DB·레시피·보유·저장)
      │                                    │
A3 뽑기 (S3: 랜덤 생성+획득 연출)          │
      │                                    │
A4 대기실 (S5: 목록·편성·뽑기 UI)          │
      └──────────────┬─────────────────────┘
A5 연결: 편성 파티 → 전투 스폰 + AI 상대 (S6 나머지+S7)
      │
A6 결과 + 씬 플로우 (S8: 루프 폐쇄)
      │
A7 알파 풀테스트 (DoD: 외부인이 설명 없이 루프 완주 + 뽑을 때마다 다른 캐릭터)
```

### 순서 논리 3가지
1. **미지수 조기 제거**: A0 스파이크 2건이 전체 구조를 좌우한다 — 합성 머티리얼이 안 되면 렌더 전략 전환(RenderTarget 굽기/소규모 풀), UMG가 안 뚫리면 UI 전략 전환. 뒤에서 터지면 재작업 폭탄.
2. **오너 가이드 대기 중 공회전 방지**: 전투 완성(A1)은 캐릭터 시스템과 독립(현행 8기 프리셋으로 진행) → A0·오너 가이드와 병렬.
3. **데이터 먼저, UI 나중**: A2(레시피·저장)가 서고 나서 A3(뽑기)·A4(대기실)가 그 위에 얹힌다. A5에서 두 갈래(캐릭터·전투)가 합류.

### 단계별 의존 메모
- A1의 HP 표시: UMG 스파이크(A0②) 결과 나오면 UMG로, 아니면 임시 월드공간 게이지 후 교체.
- A2는 **오너 가이드 + A0①** 양쪽 뒤. 가이드가 늦으면 레시피 스키마에 태그 자리만 예약하고 진행 가능(폴백).
- A5의 "편성 파티 스폰"은 BP_BattleSpawnPoint가 레시피를 받아 외형(S2)+스탯(S1)을 구성하는 접점 — 알파 최대 통합 지점.

## 4. 리스크·미지수 (스파이크로 해소할 것)
- **UMG를 MCP로 만들 수 있는가** — TextRender SetText 런타임 무반영 이력(공격버튼데모). UMG 위젯 생성·바인딩·문자열 테이블 연동이 MCP로 가능한지 미답. → A0②
- **합성 머티리얼 성능·품질** — Masked 5레이어 알파-오버 순서, 파츠 간 z 정렬, 픽셀퍼펙트 유지. → A0①
- **씬 전환 미답 영역** — 레벨 전환 시 MCP 세션·GameInstance 지속성 함정 미검증. → A6 착수 시 파일럿 프로브(§4 원칙).
- **"스킬 소수"의 범위** — balance-designer 첫 투입(A1)에서 확정(예: 강공격·힐 정도).
