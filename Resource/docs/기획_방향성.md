## 방향성 1 — 수집형 (기존, 모바일 라이브)
**캐릭터를 랜덤으로 뽑아 내 자산으로 모으고 편성해 싸우는 게임.**
- 대기실(뽑기·편성) → 전투(vs AI/PvP) → 결과 → 대기실 루프 + 로컬/서버 저장
- **캐릭터 = 영속 자산**(뽑으면 내 것, 계속 보유)
- 타겟: **모바일 라이브 서비스**
- 기존 [[로드맵_버전계획]]·[[알파_개발계획]]·[[파츠_인벤토리]]·A0 스파이크가 전부 이 방향 기준
- **오너 우려(2026-07-10)**: 1인 개발로 **작업량·라이브 서비스 대응이 어렵다** — 지속 콘텐츠 공급·과금·계정/자산 관리·고객 대응 부담.

## 방향성 2 — 드래프트 단판 (신규 검토, Steam)
**매판 새로 뽑은 캐릭터로 밴픽하고 단판 4:4로 싸우는 오락실형 PvP.**
- **1:1 PvP 유지**
- 게임 시작 → **미리 만든 ~1000개 캐릭터 중 6개 랜덤 지급**(포지션별 2개씩: 탱/딜/힐) → **5개 선택** → **상대가 1개 밴** → **4:4 배틀**
- 전투 승리 → **골드 획득** → 골드는 **캐릭터 재굴림(리롤)**에 사용(맘에 안 드는 6개를 다시 굴림). **골드는 판 간 이월, 재굴림 전용**(오너 확정 2026-07-10)
- **캐릭터 = 자산 아님.** 오락실 게임처럼 단판 플레이하고 끝, 쌓이는 자산 사실상 없음(골드만 약하게 이월)
- 타겟: **Steam**
- **리텐션**(오너 확정): Steam 리더보드(승률·승수) 연동. 랭크/MMR은 출시 성과 보고 결정(우선 제외).

### 캐릭터 1000개 = 스킬 원자 조합 생성 (오너 확정 2026-07-10 — 밸런스 실현가능성의 열쇠)
"고유 1000개"를 한땀한땀이 아니라 **정형화된 원자의 조합**으로 생성:
- **모든 캐릭터 = 평타 1 + 공용스킬 1 + 직업스킬 1** (3슬롯)
- 평타 10종 × 공용 10종 × 직업 10종 = **10³ = 1000 조합** → **설계는 스킬 원자 30개만**
- AI/스크립트로 정형 규칙에 따라 1000개 데이터 생성(parts.csv 387콤보를 스크립트로 구운 것과 동일 방식)
- 목적: 매판 다른 조합이 나와 **"다른 랜덤 도파민"** — 질리지 않는 재플레이
- **핵심 이점**: 밸런스 대상이 1000개가 아니라 **원자 30개 안팎(+조합 상호작용)** → 1인 개발로 관리 가능. 우리 [[알파_개발계획]] §2.56 **데이터 드리븐 효과 시스템**(스킬=원자 조합)과 정확히 동일 철학.

### 생성 절차 (오너 확정 2026-07-10)
1. **1000개 캐릭터 ID 선정** → 각 ID에 **포지션 배정**(탱/딜/힐 ≈ 333/333/333).
2. **포지션별 직업스킬 ≈ 4종** 제작(포지션이 직업스킬 풀을 규정) + **공용·평타 ≈ 12종** 제작(전 포지션 공유).
3. **스크립트로 1000개에 조합 랜덤 배치** → **중복 조합 검출 시 재배치**(같은 조합 2개 방지). ← parts.csv 387콤보 생성 스크립트와 **동일 패턴**(조합 생성+무결성 검증, 우리 검증된 방식).
- **외형 ↔ 포지션 느슨한 매핑**: 탱커 외형이 마법 쓰면 위화감 → 외형(무기·의상)이 포지션을 **어느정도 준수**. 의도적 예외는 허용하되 기본은 포지션 일치. → parts.csv에 **포지션 태그** 추가(방향1의 미결 `JobTags` 문제가 여기서 자연 해결 — 무기 SPEAR/WAND 등 실측 5종을 포지션에 매핑).

## Director 평가 (2026-07-10)
**방향성 2가 1인 개발 적합성에서 명확히 우세.** 근거:
- **라이브 서비스 부담 제거**: 매일 콘텐츠·이벤트·과금·고객대응·자산 분쟁이 없다. Steam 단품은 "만들어 팔고 패치"라 1인에 현실적.
- **서버 경량화**: 유저 영속 자산(계정·인벤토리·저장 무결성)을 관리 안 함 → 서버는 매칭+전투 판정만. [[알파_개발계획]] §2.56에서 확정한 "서버=경량 판정"과 더 잘 맞음.
- **우리가 방금 검증한 강점과 직결**: A0② 합성 머티리얼로 1000개 사전 생성 가능(parts.csv 387콤보 조합이면 쉬움). 기존 4v4 턴제 전투 MVP도 그대로 재사용.
- **트레이드오프**: 밸런스·리텐션·매칭에 방향1과 다른 종류의 난제(아래 열린 질문).

## 방향성 2 — 확정된 것 (오너 2026-07-10)
- ✅ **밸런스 모델**: 스킬 원자 조합. 포지션별 직업스킬≈4 + 공용·평타≈12 → 스크립트로 1000 조합 생성+중복 재배치. 밸런스 대상 = 원자 ~28개+조합 상호작용.
- ✅ **포지션**: 캐릭터 ID에 탱/딜/힐 배정(≈333씩). 직업스킬은 포지션이 규정.
- ✅ **스킬 공유**: 평타·공용은 **전 포지션 공유**(탱이 딜 평타 가능 = 조합 다양성↑).
- ✅ **외형↔포지션**: 느슨한 매핑(기본 준수+의도적 예외 허용). parts.csv에 포지션 태그.
- ✅ **골드**: 판 간 이월, 재굴림 전용.
- ✅ **리텐션**: Steam 리더보드(승률·승수). 랭크/MMR은 출시 후 결정(우선 제외).

## 방향성 2 — 다음 열린 질문 (남은 것)
1. **조합 여유 확인(가벼움)**: 포지션별 333개를 중복 없이 만들려면 (직업스킬×공용×평타) ≥ 333 필요. "공용·평타 12개"가 각각 12면 4×12×12=576≥333 여유 충분, 합쳐서 12면 부족할 수 있음 — 실제 스크립트 짤 때 확정.
2. **조합 상호작용 밸런스**: 원자 개별 밸런싱해도 특정 콤보가 우연히 OP/쓰레기 — 밴픽 재미의 원천이자 관리 포인트. **결정론 전투 공식(§전투완성)으로 1000 조합 자동 시뮬**하면 극단 조합 사전 검출(balance TTK 시뮬 확장).
3. **Steam PvP 매칭 유저풀**: 인디 PvP는 동접 부족으로 매칭 실패 흔함 → **봇/AI 폴백** 사실상 필수(방향1 vs-AI 재활용). 리더보드 우선이라 초기 vs-AI/비동기 대전도 후보.
4. **포지션 구성 규칙**: 6개(탱2딜2힐2)→5선택→1밴→4:4. 포지션 강제(최소 탱1힐1)가 있나, 밴으로 구성 붕괴 허용하나. 4:4 포지션 역할 정의도.
5. **캐릭터 ID 스키마**: [도메인 캐릭터][포지션 1자리][일련 3자리] 식 세그먼트 ID(방향1 규약 재활용) — 포지션이 ID에 인코딩되면 뽑기·밴픽 로직이 ID만으로 포지션 판별.
