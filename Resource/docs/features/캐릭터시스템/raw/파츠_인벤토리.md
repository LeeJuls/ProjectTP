**cloth top 레이어 구조 특이사항**: cloth 17종 중 11종(`cloth1,2,3,4,5,6,9,10,11,16,17`)은 `_top` 레이어가 캔버스 전체에서 완전히 공백(전체 알파=0) — 결함이 아니라 설계 특성으로 판단됨(케이프/큰 옷깃처럼 팔·머리 앞으로 나오는 부분이 없는 디자인은 `_bot`만으로 전체 프레임을 커버). 나머지 6종(`cloth7,8,12,13,14,15`, catalog 확인 결과 케이프/망토 계열)은 bot+top 둘 다 실사용. **`_bot`만으로도 17행 전체가 스킨과 동일한 프레임 수로 완전히 커버됨**(§3 표에서 검증) — top이 빈 cloth를 골라도 해당 의상이 특정 프레임에서 사라지는 문제는 없음.

## 5. 오너 가이드 작성 시 결정 필요 항목

1. **cloth 1~17 → 직업 매핑**: 파일명에 근거 없음. `catalog_cloth.png` 육안 참고해 실루엣 기준으로 배정 필요.
2. **weapon 1~5 명칭·직업 허용 목록**: SPEAR(4)·WAND(5)는 확정, SWORD/AXE/DAGGER(1/2/3 추정)는 오너 육안 재확인 권장. 어떤 직업이 어떤 무기를 쓸지도 결정 필요.
3. **skin 6색의 의미**: "인간 피부톤 다양성"으로만 쓸지, 일부 색(초록·청록·창백)을 "종족" 신호로 재해석할지.
4. **hair m/f 태그 강제 여부**: 리소스는 강제하지 않음(실측상 교차 조합 이상 없음) — "여성 캐릭터는 f 헤어만" 같은 규칙을 정책으로 걸지, 전 파츠 성별 무관 완전자유로 둘지.
5. **cloth top-empty 11종의 취급**: 망토/앞자락 있는 6종(7,8,12,13,14,15)과 시각적 임팩트 차이가 있음 — 직업별 배분 시 고려할지.
6. **weapon 색상 랜덤 범위**: weapon5만 색상축이 있어 무기 색 랜덤을 걸 수 있는 범위가 제한적. 나머지 무기는 "스타일 고정색"으로 받아들일지 여부.
7. **체형 축 포기 여부**: 리소스에 아예 없음 — 알파 범위에서 이 축을 포기하고 단일 체형으로 갈지, 이후 신규 구매/제작 대상에 넣을지.
8. **스킨-눈동자-헤어색 사이 "어울림" 규칙 여부**: 현재 리소스는 5개 색상축이 완전히 독립적(강제 조합 없음) — 완전 무작위로 둘지 조합 규칙(궁합표)을 추가할지.
