## 1. 근본원인 (진단 실측)

`WalkForward`가 도착 Z를 `WalkTargetLoc.Z = HomeLocation.Z`로 **무조건 대입**하고 있었다.

- 무대가 **오블리크+사선**(정면 스테이지 이후 v2-S2로 변경)이라 공격 위치의 바닥이 유닛 홈 위치의 바닥보다 **낮다** — 공격위치 바닥 Z(파티 **487.7** / 적 **501.1**) vs 유닛 홈 바닥(~**620~633**).
- 그런데 도착 Z에 홈 높이를 그대로 써서, 유닛이 낮은 바닥 위 공중에 뜬 채로 도착했다.
- **전 유닛 현상**(파티·적 양쪽). 원점 기준 부양량 110~146 / 실제 바닥 낙차 ~42(A1 기준).
- **Sprite 루트 확정** — 컴포넌트 계층은 무죄(부양은 좌표 대입 문제이지 계층/부착 문제가 아님).

## 2. 수정 (BP_BattleSpawnPoint — BeginPlay + WalkForward)

신규 멤버 변수 `WalkFootOffset`(float) 도입. "발 밑에서 바닥까지의 높이"를 홈에서 한 번 캐시해두고, 도착지에서 그 바닥에 다시 얹는 방식.

**BeginPlay** (footOffset 캐시):
```
footOffset = homeZ − (홈 바닥 아래방향 라인트레이스 Z)
   → WalkFootOffset에 저장
   미스 시 폴백 129.5 + 로그
```

**WalkForward** (Y가드 이후 · MoveComponentTo 이전에 삽입):
```
destGroundZ = 도착 XY 아래방향 라인트레이스
   · 채널 Visibility
   · 범위 2000 / −1000 (홈 Z 상대가 아닌 고정 절대값 = 최저 바닥 브래킷)
   · bIgnoreSelf = true
   · ActorsToIgnore = TurnQueue(전 유닛)
WalkTargetLoc.Z = destGroundZ + footOffset
   미스 시 폴백 homeZ + 로그
```

**WalkBack은 무변경**(홈으로 되돌아가는 것이므로 홈 Z 그대로가 정답).

## 3. qa BLOCKER 2건 반영

**(1) 트레이스가 유닛 콜리전을 바닥으로 오인하는 것 차단**
두 트레이스(홈·도착) 모두 `bIgnoreSelf=true`, 그리고 도착 트레이스는 `ActorsToIgnore=TurnQueue`. 유닛의 `ClickBox`·`Sprite`가 Visibility 채널을 막고 있어서, 접지용 아래방향 트레이스가 자기 자신이나 이웃 유닛을 "바닥"으로 오인하면 `footOffset`의 부호가 반전(−127류)돼 유닛이 오히려 땅에 매몰되는 결과가 나온다 — 이를 원천 차단.

**(2) 항등식 판정 금지 → 독립 교차검증**
"`destZ == groundZ + offset`" 같은 판정은 코드가 계산한 값을 그 계산식으로 다시 확인하는 것이라 **항상 참**(가짜 GREEN) — 깨진 수정도 통과한다. 판정 기준을 BP 내부 값이 아니라 **독립적인 `trace_world` 지상진실**과의 교차검증으로 대체.

## 4. 검증

- **groundZ 교차검증**: BP 자체 트레이스와 독립 `trace_world`가 groundZ를 소수점 3자리까지 일치 — 파티 **487.708** / 적 **501.066**.
- **footOffset 양수**(자기 충돌 없음): 103.6(파티) / 100.9(적).
- **효과(A1)**: destZ **633.453 → 591.267**(42cm 하강, 바닥+오프셋 = 정확히 접지).
- **오너 육안 확인 완료** — 발이 바닥에 닿음.
- **판정 도구 대체 근거**: `get_actor_bounds` 기반 8cm 접지 판정은 (a) 비-스프라이트 컴포넌트가 바운즈를 오염시키고 (b) 걸음 도착 창(0.4s)이 MCP 호출 지연보다 짧아 그 순간을 못 잡아 불가 → 독립 트레이스로 대체(Director 승인).
