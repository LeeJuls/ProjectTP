## 3. BP_BattleSpawnPoint 확장

### 신규 변수
- `HomeLocation`(Vector) — BeginPlay 체인 끝(`DiagVFXSetup.then`, 원래 미연결 종점)에 `GetActorLocation→SetHomeLocation` 추가.
- `AttackPointOverride`(Actor ref, instance-editable, 기본 None).
- `WalkTargetLoc`(Vector, 계산용 — plan 미명시 신규. 이유: WalkForward 커스텀 이벤트는 파라미터 추가 불가(`add_node_pin` "does not support adding pins" 확인), 로컬 변수도 EventGraph 커스텀 이벤트엔 불가(함수/디스패처 그래프 전용) → 멤버 변수로 대체).

### WalkForward (커스텀 이벤트)
1. **①AttackPointOverride**: `IsValid(AttackPointOverride)` → True: `GetActorLocation→BreakVector(X,Y)+BreakVector(HomeLocation,Z)→MakeVector(X,Y,HomeZ)→SetWalkTargetLoc`.
2. **②팀 포인트**: IsValid1 False → `IsValid(ManagerRef)` → True: `GetManagerRef→GetAttackPointForTeam(self=ManagerRef, bIsParty=GetIsParty)→IsValid(결과)` → True: 위와 동일 패턴으로 SetWalkTargetLoc.
3. **③폴백**: IsValid2 또는 IsValid3 False(fan-in) → `HomeLocation.X + SelectFloat(GetIsParty?300:-300)` → MakeVector(그값, HomeLocation.Y, HomeLocation.Z) → SetWalkTargetLoc.
4. **Y가드**: 3개 분기 모두 완료 후(then fan-in) → `GetTurnQueue(ManagerRef 경유)→ForEachLoop` → `NotEqual(Object)(자기 제외)→Branch→IsValid(순회유닛)→GetActorLocation→BreakVector(Y)→Subtract(WalkTargetLoc.Y)→Abs→LessThan(10.0)→Branch→True: WalkTargetLoc.Y+=12`.
5. **MID 애니**: `ForEachLoop.Completed → SetScalarParameterValue×3(TimeOffset=GetGameTimeInSeconds, FrameCount=8, RowIndex=2)`.
6. **로그** "WalkFwd:라벨:t=:target=" (PrintString, bPrintToScreen=false).
7. **MoveComponentTo**(Sprite, WalkTargetLoc, OverTime=0.4, bEaseOut=true).
8. **로그** "WalkArrive:라벨:t=:loc=" (MoveComponentTo.then에 연결 — latent 완료 후 실행되는 것으로 실측 확인).

### WalkBack (커스텀 이벤트, W0 골격 완성)
1. RUN 애니 3파라미터(TimeOffset=now, FrameCount=8, RowIndex=2).
2. `MoveComponentTo(Sprite, HomeLocation, 0.4, EaseOut)`.
3. `then`(완료) → idle 복귀(TimeOffset=0, FrameCount=6, RowIndex=0) + 로그 "WalkHome:라벨:loc=".

### EffectQuad 재계산
PlayAttack·TakeHit 각각의 이펙트 분기(`Sequence.then_1 → SetMaterial` 직전)에 `SetWorldLocation(EffectQuad, GetActorLocation + MakeVector(0,-80,20))` 삽입. `vector+vector` 프로모터블 연산자(`유틸리티|연산자|추가`로 생성 후 두 Vector 핀 연결로 자동 승격).

## 4. BP_BattleManager 확장

### 신규 변수
- `PartyAttackPoint`(Actor ref, instance-editable) → 레벨 인스턴스에 `AttackPoint_Party` 지정.
- `EnemyAttackPoint`(Actor ref, instance-editable) → 레벨 인스턴스에 `AttackPoint_Enemy` 지정.

### 신규 함수 — `GetAttackPointForTeam(bIsParty: bool) -> Actor`
Branch(bIsParty) → True: `LocalResult = PartyAttackPoint` / False: `LocalResult = EnemyAttackPoint` → 양쪽 fan-in → `Return LocalResult`.
**이 함수 자체가 명세 조정 사항**(plan.md "구현 중 명세 조정" 참조) — 원래 "유닛이 ManagerRef의 멤버 변수를 직접 조회"할 계획이었으나, MCP `create_node`가 "이 세션에서 방금 만든, 아직 어디서도 호출되지 않은 다른 BP의 멤버"를 그래프 노드로 생성 못 하는 제약이 실증되어(5회 이상 다른 문자열 형식 시도 후 확정), 대신 정식 함수를 만들어 그 함수 호출은 성공 가능한 형식(`Class|BPBattleManager|GetAttackPointForTeam`)으로 우회.

## 6. 게이트 자가검증 스캐폴드

Manager BeginPlay(원래 완전히 비어있음 — 스캐폴드 삽입에 안전)에 임시 구성:
```
BeginPlay → Delay(3.0) → 큐[0].WalkForward → Delay(1.5) → 큐[0].WalkBack → Delay(1.5) → 큐[1].WalkForward
```
스로틀(`bThrottleCPUWhenNotForeground`) 확인 결과 이미 `false`(변경 불필요).

StartPIE(warmup=8s) 실행 → GetLogEntries(pattern="Walk") 확인:
```

WalkArrive:SpawnPoint_Party_A1:t=3.4:loc=X=-350.000 Y=-6750.000 Z=633.453
WalkHome:SpawnPoint_Party_A1:loc=X=-1020.000 Y=-7380.000 Z=633.453
WalkFwd:SpawnPoint_Enemy_B1:t=6.4:target=X=350.000 Y=-6750.000 Z=621.328
WalkArrive:SpawnPoint_Enemy_B1:t=6.734:loc=X=350.000 Y=-6750.000 Z=621.328
```
WT-02·06·07 정상 확인. StopPIE 후 WT-01(Enemy_B1.AttackPointOverride=AttackPoint_Party 임시 세팅) 재PIE:


StopPIE 후 EnemyAttackPoint=AttackPoint_Enemy 복원 확인. 스캐폴드 노드 10개 전부 delete_node로 제거, Manager BeginPlay 원래(빈) 상태로 복귀 확인, 재컴파일 에러 0.
