# 청사진 — 카메라액션 (공격 액션 컷)

옥토패스식 컷 감성: 공격(Executing) 동안 팀별 근접 앵글로 부드럽게 전환 → 턴 끝에 기본 카메라 복귀. **액션 컷만**(쉐이크는 후속), **앵글은 오너 직접 배치**.
#projectTP/카메라액션

## 목표
- 자기 턴 공격 시작 시 해당 팀(아군/적)의 근접 액션캠으로 부드럽게 전환.
- 턴 종료 시 항상 기본 카메라로 복귀(null-skip 경로 포함 — 단일 복귀 지점).

## 범위 / 비범위
- ✅ 범위: 팀별 CameraActor 2기 배치·기본 앵글 산출, `SetViewTargetWithBlend` 배선(EnterExecuting 컷·EnterTurnEnd 복귀), 캠 null 시 우아한 폴백.
- ❌ 비범위: 카메라 쉐이크, 턴 시작 팬, 스킬 클로즈업, 타겟 리액션 컷(피격자 앵글), DoF·시네마틱 카메라.

## 큰 흐름
```
EnterExecuting(HideAll 직후, WalkForward 앞)
  → GetPlayerController(0) → IsValid
     [null] → WalkForward(기존 체인 그대로 진행, 컷 스킵)
     [valid] → Select(ActiveUnit.bIsParty ? ActionCamParty : ActionCamEnemy) → IsValid
        [null] → WalkForward(스킵)
        [valid] → SetViewTargetWithBlend(0.4s, EaseInOut) → PrintString(CamCut 로그) → WalkForward(정상 진행)

EnterTurnEnd(진입 최초, 마커OFF보다 먼저)
  → GetPlayerController(0) → IsValid
     [null] → SetBattleState(기존 체인 그대로)
     [valid] → SetViewTargetWithBlend(DefaultCamera, 0.3s, EaseInOut) → PrintString(CamBack 로그) → SetBattleState(정상 진행)
```

## 관련 에이전트
- Director(Fable) — 계획·앵글 시작값 산출·최종 게이트 판정.
- gameplay-engineer(sonnet) — C0(배치+변수)·C1(배선+게이트) 구현.
- 오너 — CF 단계에서 ActionCam 2기 직접 드래그·회전으로 앵글 튜닝.
