## 카메라 액터·변수
- `ActionCam_Dynamic` CameraActor 1기 배치(`CameraActor_3`, 폴더 `BattleStage/Camera`, FOV=55). 초기 위치는 BattleCamera 근접 좌표(0,-7850,750) — Manager가 매 공격마다 SetActorLocation/Rotation으로 덮어쓰므로 초기값은 무관.
- Manager 변수: `ActionCamDynamic`(Actor, instance-editable) + float 6종(`CamBack`/`CamLateral`/`CamHeight`/`CamLookBias`/`LookAtZOffset`/`CamBlendIn`, 전부 instance-editable).
- **CDO 개별 set_properties×6 + 레벨 인스턴스도 별도 세팅 필요**(신규 함정 발견, §언리얼_MCP_실전노하우 갱신): CDO에만 세팅하면 기존 레벨 인스턴스(`BP_BattleManager_C_0`)의 float 값이 0으로 남는다(변수 신규 추가 시 인스턴스가 CDO 기본값을 자동 상속하지 않는 것으로 관찰됨). CDO·인스턴스 양쪽에 개별 set_properties + 매회 get_properties 재확인이 필요.
- 확정값: `CamBack=350`/`CamLateral=250`/`CamHeight=250`/`CamLookBias=0.7`/`LookAtZOffset=120`/`CamBlendIn=0.25`.

## OTS 계산 배선 (~54개 노드)
`EnterExecuting`(`PrintString_169.then`)에 삽입, 정상/스킵 경로 전부 `WalkForward.execute`로 fan-in(총 6개 fan-in: IsValid×4 스킵 + Length<1cm 스킵 + 정상완료).

```
가드: IsValid(PC)→IsValid(ActiveUnit)→IsValid(SelectedTarget)→IsValid(ActionCamDynamic)→Branch(Length(T-A)<1cm)
계산: A/T=GetActorLocation, axis=Normalize(T-A), up=(0,0,1), cross=CrossProduct(axis,up), lat_raw=Normalize(cross)
      latY=BreakVector(lat_raw).Y, gt_zero=(latY>0.0)
      flipped=(0,0,0)-lat_raw, lat_final=SelectVector(A=lat_raw, B=flipped, bPickA=NOT(gt_zero))
      camLoc = A - axis*CamBack + lat_final*CamLateral + (0,0,CamHeight)
      lookAt = Lerp(A,T,CamLookBias) + (0,0,LookAtZOffset)
적용: SetActorLocation(ActionCamDynamic, camLoc) → SetActorRotation(ActionCamDynamic, FindLookatRotation(camLoc,lookAt))
      → SetViewTargetWithBlend(PC, ActionCamDynamic, CamBlendIn, EaseInOut) → PrintString(log) → WalkForward
```

## 로그 형식 확장 (오너 보충 지시 반영)
plan 원안 로그는 `"CamCut:A={0}:T={1}:cam={2}"`(A/T/cam만)였으나, 오너가 "게이트 스캐폴드 판정 시 camLoc·lookAt 기대값 계산을 위해 6수치와 A·T 좌표를 함께 출력"하도록 명시 지시 → `add_node_pin`×6으로 FormatText 인자 확장, `"CamCut:A={0}:T={1}:cam={2}:back={3}:lat={4}:height={5}:bias={6}:zoff={7}:blend={8}"`로 변경. 6수치 인자는 camLoc 계산에 이미 쓰인 pure 게터 노드를 fan-out 재사용(연결만 추가, 신규 게터 불필요).
