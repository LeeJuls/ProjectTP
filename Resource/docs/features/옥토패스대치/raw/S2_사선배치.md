### 6-2. PNG 파일화 — 이 세션 도구 제약으로 미완료
CaptureViewport의 base64 이미지(각 약 618만자)가 MCP 응답 토큰 한도를 초과해 하니스가 자동으로 tool-results txt 파일로 저장하는 동작이 재현됨(P1 문서와 동일 현상). 지시서가 지정한 `python decode_capture.py <txt> <png>`를 실행하려 했으나, **이 세션에는 Bash/셸 실행 도구가 제공되지 않아 Python 스크립트를 실행할 방법이 없음**을 확인했다(ToolSearch로 bash/shell/terminal 키워드 검색 결과 없음).

우회를 위해 `ProgrammaticToolset.execute_tool_script`(sandboxed Python, 허용 모듈 `json/math/datetime/copy/re/time`만)로 CaptureViewport를 호출하고 반환된 base64를 스크립트 내부에서 직접 파일에 써서 에이전트 컨텍스트를 거치지 않게 시도 — 이 부분은 성공(`AssetTools.write_file`로 프로젝트 Saved 폴더에 base64 텍스트로 저장 완료, 아래 경로 참조). 그러나 **base64→PNG 바이너리 디코딩 자체는 이 샌드박스에 `base64` 모듈이 없고 `write_file`이 텍스트 전용이라 이 세션 내에서 완결 불가**로 확인됨. `TextureTools.import_file`도 "디스크에 이미 존재하는 이미지 파일"을 가정하므로 우회 경로가 되지 못함.

**현재 상태(중간 산출물, 다음 세션/Bash 가능 환경에서 마무리 필요)**:

- `D:/unreal/projectTP/Saved/S2_B_b64.txt` — B안 참고 캡처의 base64 PNG 텍스트(616만0980자)
- 마무리 절차: 위 두 txt를 `decode_capture.py`(또는 동등한 base64 디코드)로 디코딩 → `docs/features/옥토패스대치/raw/S2_A안.png`, `raw/S2_B안참고.png`로 저장.

이 건은 CLAUDE.md 원칙("3회 실패 시 중단→오너 호출", "무한 시도 금지")에 따라 추가 시도를 중단하고 여기서 명확히 보고한다. 카메라 값·씬 상태 자체는 정확히 세팅·검증되었으며, 막힌 것은 순수히 "PNG 바이트 실체화" 단계다.
