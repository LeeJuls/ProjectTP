## 복원 절차 (오너가 "배치_1 불러와줘" 요청 시)

1. 아웃라이너 `BattleStage/SpawnPoints` 폴더에서 8개 라벨(`SpawnPoint_Party_A1..A4`, `SpawnPoint_Enemy_B1..B4`)로 각 액터를 식별.
2. 각 액터에 `ObjectTools.set_properties`(또는 Details 패널 수동 입력)로 위 표의 `Location(X,Y,Z)`을 재적용. Rotation/Scale은 root가 identity 그대로이므로 건드리지 않는다.
3. 각 액터에 `faceLeft` 값을 위 표대로 재적용. **FaceLeft가 바뀌면 Construction Script가 자동으로 SpriteRight/SpriteLeft MI 전환 및 관련 시각 갱신을 수행**하므로 MI 항목은 직접 만질 필요 없음(표의 MI 열은 검증·참고용).
4. 8기 전부 적용 후 `get_actor_transform` + `get_properties(["faceLeft"])`로 재조회해 이 표와 일치하는지 확인.
5. 변경이 실제로 있었다면 `save_assets(["/Game/Stages/map_battle_octopath"])`로 저장.

이 문서 하나만으로 gameplay-engineer/scene-builder 누구든 복원 가능하도록 절차를 자기완결적으로 기술함.

## 참고 (배치_1 시점 씬 상태 — 캐릭터 8기 외 컨텍스트, 비교 대상 아님)

- 배경 나무: `InstancedFoliageActor_0`의 `FoliageInstancedStaticMeshComponent_0`(나무)가 2회 제거를 거쳐 524 → 514그루 상태([[나무제거_백업]] 참고). 배치_1은 이 나무 개수에 의존하지 않음.
- 카메라: `BattleCamera`/`CameraActor_0`, (229.4, -8850.8, 1237.1), yaw=84° — A안. 8기를 depth 1340~2080cm·lateral -666~+1118cm 범위에 담는 실전투 카메라로 추정(레벨 내 별도 `Camera`(CineCameraActor)는 전투 뷰 아님, [[나무제거_백업]] 참고 발견 항목 참조).
- 이 스냅샷 조회는 **읽기 전용**(get_actor_transform / get_properties만 사용)이었고, 8기 트랜스폼·FaceLeft는 조회 전후 변경 없음. `map_battle_octopath`는 이 작업으로 dirty 상태가 되지 않았다(확인 완료).

## 네이티브 스냅샷 도구 조사 결과

- MCP unreal-mcp 전체 툴셋(`list_toolsets` 15개: AgentSkill/ConfigSettings/EditorApp/Logs/Plugin/Actor/Asset/Blueprint/CurveTable/DataAsset/DataTable/Material/MaterialInstance/Object/Primitive/Scene/SkeletalMesh/StaticMesh/StringTable/Programmatic/Texture) 중 Level Variant Sets, Level Snapshots, 또는 그에 준하는 액터 상태 캡처/복원 전용 툴셋은 **없음**.
- 엔진 자체에는 Variant Manager 관련 클래스(`/Script/VariantManagerContent.LevelVariantSets`, `VariantSet`, `LevelVariantSetsActor` 등)가 플러그인으로 존재함을 `search_subclasses`로 확인했으나, 이를 **authoring**(Variant 추가·프로퍼티 캡처 바인딩 구성)하는 MCP 도구는 노출되어 있지 않다. `ObjectTools.set_properties`로 임의 UObject 프로퍼티는 세팅 가능하지만, VariantManagerContent 내부의 저수준 캡처 바인딩 구조를 몇 번의 호출로 정확히 구성하는 것은 신뢰도가 낮고 깨진 애셋을 남길 위험이 있어 시도하지 않음(지시서 기준: "불확실하면 시도하지 말 것").
- 결론: 네이티브 애셋 기반 스냅샷은 **만들지 않음**. 이 문서(라이브 실측값 + 복원 절차)가 유일하고 가장 신뢰할 수 있는 복원 수단이다.
