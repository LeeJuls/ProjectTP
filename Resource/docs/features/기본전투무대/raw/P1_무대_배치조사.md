## 조사 방법
- MCP `SceneTools.find_actors`(bounds 필터) + `ActorTools.get_actor_transform/get_actor_bounds` + `SceneTools.trace_world`(수직 레이캐스트로 지면 Z 실측)로 지형·프롭 배치 파악.
- `EditorAppToolset.WorldPosToScreenCoords`로 후보 카메라의 실제 화면 프레이밍 검증(뷰포트 캡처는 이미지가 6MB+로 파이프라인 한계 초과 → 좌표 기반 검증으로 대체).
- `StaticMeshTools.get_bounds`로 `/Game/Meshes/SM_SpriteQuad` 로컬 바운드 확인.
- 원본 레벨은 읽기 전용으로만 로드(`load_level`은 조회 목적). 액터 생성/이동/삭제 없음, 저장 안 함.

### 쿼드 스케일 기준 (셀 100×40 규약)
- `/Game/Meshes/SM_SpriteQuad` 로컬 바운드 실측: X[-50,50], Y[-50,50], Z≈0 (100×100 평면, 두께 없음).
- heroes99 셀 규격: 100×40px, 캐릭터 실루엣 실측 bbox(idle1) ≈ 23×27px(셀 내부).
- **제안 규약: 1px = 1cm, 쿼드를 "셀 전체(100×40cm)" 크기로 스케일링** → 스케일 X=1.0, Z=0.4(Y=1.0 두께 無, roll=90 세워서 사용, S0 관례 계승).
  - 알파 마스크가 셀 여백을 투명 처리하므로 쿼드 자체는 셀 크기, 실제 보이는 실루엣은 그 안 23×27px.
- **캐릭터를 실제 성인 체구(약 175cm 높이)로 보이게 하려면 추가 배율 필요**: 175cm ÷ (27px 유효높이 × 1cm/px) ≈ 6.48배. 즉 최종 스케일 ≈ X=6.48, Z=6.48×0.4=2.59 (또는 셀 기준 그대로 X=6.48, Z=2.59) — **S0에서 임의로 썼던 (3.75, 1.5, 1)보다 큰 값**. S0는 검증용이라 스케일 규약이 없었음(작업로그 §10 "바꿀 것" 항목). **이번 4v4는 명시적 규약(1px=1cm 기준 + 성인 175cm 배율)을 채택 제안** — 최종 확정은 art-pipeline/hd2d-art-director와 합의 필요.
  - 참고: 스케일을 크게 잡으면(6.48배) 캐릭터 폭도 100cm×6.48=648cm급으로 과도하게 넓어질 수 있음(셀 여백 포함이라 실제 실루엣 폭은 그보다 훨씬 좁긴 함, 23px×6.48≈149cm 정도 어깨폭 추정). **балance-designer/art-pipeline과 최종 스케일 합의 필요**, 본 조사는 산출 근거 제공까지.

## (c) 카메라 초기값 제안

### 검증 방법
`WorldPosToScreenCoords`로 8기 배치 라인 양 끝(X=±600, Y=-6600)의 발밑(Z=486)·머리(Z=661, 175cm 가정) 화면 좌표를 실측(현재 뷰포트 기본 FOV 기준).

### 안 1 — 즉시 검증됨 (뷰포트 기본 FOV, 약 90°)
```
location: (X=0, Y=-7550, Z=750)
rotation: (pitch=-6, yaw=90, roll=0)
```
실측 화면 좌표(0,0=좌상단, 1,1=우하단):
| 지점 | 화면 X | 화면 Y |
|---|---|---|
| 좌끝 발밑(-600,-6600,486) | 0.809 | 0.643 |
| 좌끝 머리(-600,-6600,661) | 0.814 | 0.490 |
| 우끝 발밑(600,-6600,486) | 0.191 | 0.643 |
| 우끝 머리(600,-6600,661) | 0.186 | 0.490 |
| 중앙 발밑(0,-6600,486) | 0.500 | 0.643 |

좌우 대칭 정확(0.191/0.809, 중앙 0.500), 좌우 여백 약 19%씩 확보, 상하 여백도 충분(발밑 0.643, 머리 0.490 — 화면 중하단부에 위치, 지면/하늘 여백 넉넉). **이 값 그대로 사용 가능.**

### 안 2 — HD-2D 표준 좁은 FOV(50°) 가정 시 권장 대안
FOV를 90°→50°로 좁히면(약한 원근이지만 완전 정사영은 아닌 절충값) 동일 프레이밍 유지를 위해 카메라를 더 후퇴시켜야 함(계산상 거리 약 1.7배):
```
location: (X=0, Y=-8210, Z=950)
rotation: (pitch≈-10.2, yaw=90, roll=0)
FOV: 50 (기존 MCP SetCameraTransform에 FOV 필드 없음 — CameraActor/CameraComponent의 FieldOfView 프로퍼티로 별도 설정 필요, 구현 단계에서 처리)
```
지면 Z(0,-8210) 실측 = 512.71 (참고용, 카메라는 공중 배치라 무관).
**주의**: 이 안은 계산값이며 FOV=50 세팅 상태에서 재검증(WorldPosToScreenCoords 재확인) 안 함 — MCP 툴에 뷰포트 FOV 직접 세팅 함수가 노출되지 않아 실증 제한. 구현 단계(scene-builder 실배치 시)에서 CameraActor 생성 후 FieldOfView 설정하고 재검증 필요.

### 권장
MVP 1차 구현은 **안 1(뷰포트 기본 FOV, 실증 완료)**로 시작 → hd2d-art-director가 룩 패스에서 FOV/틸트시프트 조정 시 안 2 방향(좁은 FOV+후퇴)으로 미세조정 권장.
