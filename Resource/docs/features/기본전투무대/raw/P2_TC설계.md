## 사전 실측(TC 근거 확정)

qa-critic가 원본 에셋으로 직접 확인한 사실(TC의 기대값 출처):

- 합성 시트 `hero_knight.png` = **800×680**, 8열/17행 → 셀 정확히 **100×40**(W/8=100.0, H/17=40.0, 나눠떨어짐).
- Row 0 셀 bbox: **col 0~5 내용 있음, col 6·7 = None(빈 셀)**. → idle 정확히 6프레임, GridX=8과 FrameCount=6 **불일치가 정상**. 이 갭이 SubUV 최대 함정.
- weapon5만 `weapon5_c1~c4_bot/top.png` 색상변형 존재. weapon1~4는 `weaponN_bot.png` 단색(색인 없음). **B1만 색상변형 경로**.
- idle1 실루엣 bbox(셀 내부) = **23×27px** (셀 100×40 안에 치우쳐 존재, col0 기준 x[29,52] y[7,34]).
- 셀 비율 100:40 = **2.5:1**. 쿼드 `SM_SpriteQuad` 로컬바운드 100×100(정사각). → **유니폼 스케일 시 세로 2.5배 눌림/늘어남**.
- SubUV wrap 검산: FrameCount=6이면 frame∈0~5 → col 0~5만 참조(정상). FrameCount=8(=GridX 오설정)이면 tick6·7이 **col 6·7(빈 셀) 참조 → 빈 프레임 깜빡임**. 이 오설정이 가장 흔한 실수라 별도 TC로 격리.

## 교차·회귀 주의(단계 경계 함정)

- **S2↔S4 연동**: S2[05] FrameCount 오설정은 S2 정지 검증(frame0만 봄)에선 안 잡히고 **S4에서만 빈 프레임 깜빡임으로 발현**. → S2 게이트 통과해도 S4[02]에서 반드시 재검증.
- **S1↔S2↔S3 좌표 사슬**: 시트가 800×680 아니면(S1[02]) → UE 셀 UV 어긋남(S2[03]) → 프레임 잘림(S4). 상류 실패가 하류에서 다른 증상으로 위장.
- **스케일 미확정 리스크**: 절대 스케일은 오너 확정(S3 스크린샷 후). 비율 2.5:1만 자동 검증 가능. 절대크기 관련 겹침(S3[09])은 스케일 확정 후 재검증 필요 → **이월 TC 후보**.
- **yaw 정면 응시(S3[06])**: P1 무대조사 메모에 "마주보기 vs 정면응시" 미확정으로 남았으나 plan §5는 "전원 카메라 정면 응시"로 확정. 배치가 이 확정을 따르는지 검증(마주보기로 잘못 구현 시 한 팀 뒤집힘).
