## 1. 결론 요약 (핵심)

**글리치(가로 스미어) 미해결.** 지시서가 제시한 가설(SubUV UV이동을 모션블러/velocity가 스미어)을 포함해 **렌더링 파이프라인 전 영역을 체계적으로 배제 조사**했으나 원인을 특정하지 못했다. 아래 "배제 목록"의 항목을 하나씩 바꿔가며 캡처 비교했지만 **단 하나도 스미어를 없애거나 유의미하게 변화시키지 못함**. 3회 이상 실패한 가설 후 중단 규칙에 따라, 추가 진단은 오너 판단(또는 cpp-engineer/렌더링 전문 인력 투입)이 필요하다고 판단해 보고한다.

룩 패스(AutoExposure/Bloom/Vignette/Saturation/Contrast)는 지시대로 완료했다. 글리치와 무관하게 적용 가능한 부분이라 진행했다.

### 관찰된 추가 단서 (미해결이나 기록)
- FXAA·TSR 두 캡처를 픽셀 diff한 결과 20%가량 픽셀에서 5 이상 차이(`max diff 182`) — **완전히 결정론적인 고정 패턴은 아니고 프레임마다 미세하게 변하는 temporal한 요소가 있음**. 그러나 GI/AO/Reflection을 모두 꺼도 무효였으므로 이 temporal 변화의 정확한 소스는 특정 못함(단순 캡처 인코딩 오차일 가능성도 배제 못함).
- 스미어 패턴은 **셀 경계와 무관하게 캐릭터 실루엣 전체에 걸쳐 조밀한 가로줄**(간격 3~8px)로 나타나며, 줄의 색상(자주/하늘색/주황)이 해당 캐릭터 원본 텍스처 색상(어두운 보라/초록/빨강)과 **일치하지 않음** — 다른 소스에서 색이 섞여 들어오는 것처럼 보이나 반사를 꺼도 무효였으므로 출처 불명.
- 저채도·어두운 캐릭터(B3-검정/초록, A2-갈색, B4 등)에서 뚜렷, 고채도·밝은 캐릭터(A4-분홍, B2-빨강)에서는 거의 안 보임 — 이 색상 상관관계는 S3부터 일관되게 재현됨.

### 다음 조치 제안 (오너 판단 필요)
- **A안**: `velocityPass=AfterBasePass` 적용 후 **에디터 재시작**해서 재검증(이번 세션에서 불가능했던 유일한 미검증 항목).
- **B안**: 스프라이트 머티리얼을 Masked 대신 **Opaque + 별도 알파 블렌드 방식**으로 재설계해 비교(구조적으로 다른 블렌드 모드 자체가 원인일 가능성 — 이번 세션에서 미시도).
- **C안**: cpp-engineer 또는 렌더링 전문 인력에게 RenderDoc 등 GPU 프레임 캡처 도구로 실제 draw call/버퍼 단위 진단 의뢰(MCP 도구로는 셰이더 파이프라인 중간 버퍼를 볼 수 없어 이 이상의 진단이 어려움).
- **D안**: 별도 프로젝트(빈 템플릿)에 동일 SubUV 머티리얼 + 동일 텍스처로 최소 재현 케이스를 만들어 다른 변수(맵의 다른 액터, 조명 등) 배제 후 재진단.

## 3. 룩 패스 (완료)

지식베이스 실측값 기준, 이 맵(`map_village_day` 복제 — 자체 하늘/라이트/기존 색보정 보유)에 맞춰 미세조정.

### 적용값 (현재 PPV_Unbound 상태)
| 항목 | 원래값 | 적용값 | 비고 |
|---|---|---|---|
| AutoExposure Min/Max Brightness | override 없음 (`-0.5`/`0.5`, 자동) | **1.0 / 1.0** (override on) | 노출 흔들림 제거. 워시아웃 없음 확인 |
| Bloom Intensity | `4.0` (override on) | **0.45** | 4.0은 과다 — 하이라이트 번짐 심함. 0.45로 대비 회복 |
| Vignette Intensity | override 없음(`0.4`, 미적용) | **0.45** (override on) | 가장자리 감광, 과하지 않음 |
| Color Saturation | override 없음(`1.0`) | **1.0** (override on, B안) / 1.15(A안) | 이 맵 자체가 이미 채도 높은 목재/오렌지 팔레트라 1.15는 과포화(A안). 1.0 유지가 무난(B안) |
| Color Contrast | override 없음(`1.0`) | **1.03** (override on, B안) / 1.08(A안) | 1.08은 톤이 다소 과했음(A안). 1.03이 무난 |
| Motion Blur Amount | `0.5`(엔진 기본, override off) | **0** (override on) | 글리치 대응 1단계 조치. 글리치 해결 효과는 없었으나 HD-2D 스타일상 모션블러 자체는 불필요하므로 유지 권장 |

### Before / After 스크린샷
- Before(원본, 룩 패스 전): `raw/S5_look_before.png`
- After A안(지식베이스 값 그대로 — saturation 1.15/contrast 1.08, 채도 다소 강함): `raw/S5_look_A_saturated.png`
- After B안(완화 — saturation 1.0/contrast 1.03, 이 맵에 더 무난): `raw/S5_look_B_mild.png` ← **현재 PPV 적용 상태**

### 오너 선택 필요
- **A안 vs B안**: A안은 지식베이스 표준값 그대로라 다른 맵과의 일관성이 있지만 이 맵(목재/오렌지 팔레트 강한 village 맵)에서는 다소 과포화. B안은 이 맵 전용으로 완화한 값. **오너가 스크린샷 비교 후 택1 또는 절충값 지정 요청.**
- 틸트시프트 DoF는 이번 세션에서 미적용(지시서상 "선택" 항목, 글리치 미해결 상태에서 추가 시각 변수를 넣지 않기 위해 보류). 오너 승인 시 다음 세션에서 추가 가능.
