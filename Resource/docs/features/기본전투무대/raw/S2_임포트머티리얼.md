## 2. 마스터 머티리얼 `/Game/Materials/M_Sprite_Flipbook_Lit`

`M_Sprite_Lit`(S0, Masked/TwoSided/DefaultLit, TextureSampleParameter2D RGB→BaseColor·A→OpacityMask) 구조를 계승해 신규 생성 후 SubUV 그래프 확장.

### 그래프 로직
```
frameIndex = floor(Time × FPS) mod FrameCount        (Row0 6프레임 순환)
col        = frameIndex mod GridX
U = (baseU(TexCoord.R) + col) / GridX
V = (baseV(TexCoord.G) + RowIndex) / GridY
UV = AppendVector(U, V) → TextureSampleParameter2D(SpriteTex).UVs
SpriteTex.RGB → MP_BaseColor, SpriteTex.A → MP_OpacityMask
```
노드: Time, Multiply(×FPS), Floor, Fmod(mod FrameCount), Fmod(mod GridX), TextureCoordinate, ComponentMask×2(baseU/baseV), Add×2, Divide×2, AppendVector, TextureSampleParameter2D(SpriteTex).

### 파라미터(스칼라 5 + 텍스처 1)
| 이름 | 타입 | 기본값 |
|---|---|---|
| GridX | Scalar | 8 |
| GridY | Scalar | 17 |
| RowIndex | Scalar | 0 |
| FrameCount | Scalar | **6** |
| FPS | Scalar | 8 |
| SpriteTex | Texture | (MI에서 오버라이드) |

### 머티리얼 프로퍼티 검증
```json
{"blendMode":"BLEND_Masked","shadingModel":"MSM_DefaultLit","twoSided":true}
```
`MP_BaseColor` ← TextureSampleParameter2D_0.RGB, `MP_OpacityMask` ← TextureSampleParameter2D_0.A (get_property_input로 실측 확인).

`recompile()` 셰이더 컴파일 에러 없이 통과.

→ **[S2][04] PASS**(Masked/TwoSided/Lit, RGB→BaseColor, A→OpacityMask), **[S2][05] PASS**(GridX8/GridY17/RowIndex0/**FrameCount6**/FPS8 — 8이 아닌 6으로 정확히 설정, plan.md 경고사항 반영 완료).

## 4. frame0 정지 표시 검증

### 시행착오 (근본원인 기록)
1차 시도: `SM_SpriteQuad`에 `rotation=(pitch0,yaw0,roll0)`으로 배치 → 캡처 결과 스프라이트가 안 보임.
- **원인**: `SM_SpriteQuad` 로컬 바운드가 `(-50,-50,~0) ~ (50,50,~0)` — 쿼드가 로컬 **XY 평면**(Z≈0, 두께 없음)에 누워 있는 형태. `roll=0`으로 두면 쿼드가 수평으로 누워 카메라 정면(yaw=90 방향)에서 볼 때 선처럼 얇게만 보임.
- **가설 검증**: 참고용 `/Game/test` 레벨의 기존 `SM_SpriteQuad` 사용 액터 트랜스폼을 조회 → `rotation=(pitch0,yaw0,roll=90)` 확인. `roll=90`으로 회전해야 쿼드가 수직으로 서서 카메라를 정면으로 마주봄.
- **수정**: `rotation=(0,0,90)`으로 재배치 → `get_actor_bounds`가 `z: 650~750`(두께 발생, 수직 확인) 반환 → 캡처 시 캐릭터 스프라이트 정상 표시.

### 검증 방법
- 원본 `map_village_day` 레벨(무오염 유지 목적)에 임시 액터 `TEMP_S2_Verify_Quad`(`SM_SpriteQuad`) 배치.
- **주의**: `StaticMeshTools.set_material`은 메시 에셋 자체의 슬롯을 영구 변경하므로 사용 금지(1차 시도에서 실수로 사용 → 즉시 `get_referencers`로 원래 값(`M_Sprite_Lit`) 확인 후 원복). 대신 `ObjectTools.set_properties`로 `StaticMeshComponent.overrideMaterials`(컴포넌트 단위 오버라이드)를 사용해 `MI_Party_A1` 적용 — 메시 에셋은 불변.
- 검증 후 임시 액터 `remove_from_scene`으로 제거, 메시 슬롯 원상태(`M_Sprite_Lit`) 재확인, 레벨은 **저장하지 않음**(dirty 상태로 남지만 디스크 원본 무오염).
- 신규 에셋(텍스처 8 + 마스터 머티리얼 + MI 8) 17개만 `save_assets`로 명시적 저장.
