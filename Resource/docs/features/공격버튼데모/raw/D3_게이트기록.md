## 판정 히스토리 (오판→정정 기록 — 교훈 보존용)
1. 1차 실클릭에서 "복귀 실패"로 오판 → 수정 에이전트의 정적 분석(배선 전 구간 정상)과 모순 → **idle 레퍼런스([[D1_octopath_idle.png]]) 대조로 A1의 idle 자체가 웅크린 스탠스임을 확인, 오판 철회**. 복귀는 처음부터 정상 작동. → 교훈은 [[../../../언리얼_MCP_실전노하우|실전노하우 §6]]에 기록.
2. "에디터가 TextRender 미표시" 추정(D2)도 오진 — 실원인은 Label 회전(법선이 하늘 향함)+WorldSize 과소. 회전(pitch90,yaw0,roll180)·WorldSize 50·RelativeLocation(40,0,72)로 해결.
3. Label 내용은 런타임 SetText 무반영(원인 미규명) → **에디터 프로퍼티에 LOCTABLE("/Game/UI/ST_UI","Battle.Attack") 임포트로 우회** (1안 성공).

## 미결·부채 (후속)
- SetText(TextRender) 런타임 무반영 원인 미규명 — 데모엔 영향 없음(에디터 프로퍼티로 확정). 전투 UI 본격 개발 시 UMG로 대체 예정이라 실害 낮음.
- LOCTABLE 참조의 동적 유지(언어 전환 시) 미검증 — 로컬라이제이션 도입 시 재확인.
- 공격 체감 속도(현 1.0s) 튜닝은 오너 체감 후(FPS 상향 시 RevertToIdle에 FPS 복원 추가 필요 — plan 명세 참조).
- 버튼 앞 전경 나무 말뚝 부분 겹침 — 기존 배경 요소. 거슬리면 버튼 위치 이동 또는 말뚝 제거(오너 판단).
