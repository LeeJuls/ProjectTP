# 전투완성(A1) 청사진

> 상위: [[알파_개발계획]] flow A1 · 설계 근거: [[스탯_전투공식_v1]](qa 조건부 GO)+[[모션연결_규칙안]](E1-B 오너 확정) · 승인 계획 원문: Claude plan 파일(v3, git 외부) — 요지는 이 문서와 [[plan]]에 전사.
#projectTP/전투완성

## 목표
현행 턴제전투MVP(핫시트 4v4, 애니메이션 싱크만)를 **진짜 전투**로 완성한다: HP·데미지·사망·승패 + 스킬 5종 + 모션 배선(E1 이관) + 광폭화(F1 게이트 조건부).

## 범위 / 비범위
- **범위**: 스탯 로드(CSV→DT), TakeHit §8 판정(0~9, **8.5 상태이상 확률판정 포함**), 사망·승패(End 상태), 스킬 5종(공격/베기/파이어볼/막기/치유)+쿨다운+**상태이상 2종(기절 STUN·공격력약화 ATK_DOWN, 확률 발동 — [[상태이상_확정]])**, 모션 배선(RowIndex/FrameCount 데이터 구동), 광폭화(30턴+5% 가산), 임시 월드공간 HP게이지·스킬슬롯.
- **비범위**: AI 상대(→A5), Result 화면·재시작 플로우(→A6, End는 BattleFinished 디스패처 방출까지), WBP_BattleHUD(→별도 명세 system-ui #1 — 임시물은 thin·`_A1Temp` 격리·스코프 펜스), 30원자 스킬 재편(→balance#1/A2), 데미지 부동숫자·턴순서바·ATB(임시물 금지 목록), **실제 AoE(전체공격) 스킬 — `ENEMY_ALL`/`ALLY_ALL` 토큰 발급·`AwaitTarget` ALL모드·AoE 스킬행·PowerRate 밸런스는 A2**(구조 계약 — SelectedTargets 배열화·ResolveTargetPool 단일소스 함수·토큰 예약 — 만 A1에서 선병합, [[상태이상_확정]] §11).

## 큰 흐름 (스테이지 F0~F9)
F0 선결확정(8건: 타겟팅 분기표·배정표·motions 대분류6·DYING/BLOCK 정지 메커니즘·EndBehavior 컬럼·스캐폴드 인터페이스 등) → F1 광폭화 재검증(balance, 병렬, F8 게이트) → F2 데이터 부트스트랩(오너 struct 3종+DT 파일럿→확장) → F3 스탯+HP게이지 → F4 TakeHit 코어(전체 스켈레톤) → F5 사망·승패(End=EnterTurnEnd 단일 관문 — 레이스 차단) → F6 모션 배선(유지시간 `(FrameCount/8)−0.05` 데이터 구동) → F7 스킬 확장(매니저 소유 원칙·슬롯 동적 재바인딩) → F8 광폭화(step4 국소·스탯 불변·양측 대칭) → F9 풀회귀(F9a 기계 원장대조 / F9b 오너 육안).

## 핵심 리스크 (리뷰에서 확정된 것)
1. **End 상태 레이스**: Executing 병렬 B분기(WalkBack→TurnEnd)가 End를 덮음 — EnterTurnEnd 단일 관문+bBattleOver 가드로 차단(F5).
2. **DYING/BLOCK 정지 불가**: 셰이더 모듈러 수식(`floor((Time−TimeOffset)×8)%FrameCount`)에 hold 없음 — 마스터 머티리얼 Freeze 파라미터 신설(F0⑥, art-pipeline)로 해결, 폴백=알파 한정 루프 허용.
3. **CAST/BLOCK 리와인드 팝**: FrameCount5 행이 0.70s 유지 시 frame0 재노출 — 유지시간 데이터 구동으로 원천 제거(F6).
4. **최초 경로 2건**: DataTable e2e 임포트+CSV bool 컬럼(F2 파일럿로 위험 격리).
5. **상태이상 난수 최초 도입**: 결정론 정답지(F9a)가 step8.5 하나에서 처음 깨짐 — 검증모드 훅(전역 Chance 0/1 강제)으로 L0/L1 복원, 실플레이는 로그(effectRoll 사후감사)로 L2 결정론 대조([[상태이상_확정]] §12).
