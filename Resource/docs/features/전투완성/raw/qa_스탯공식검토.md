## 2. 중(High) 상세

### H1 — 막기 DMG_REDUCE 의미 미확정 (모순+명세공백)
- **근거(인용)**: §8 "DMG_REDUCE는 … **피격 시 소비 또는 턴 경과로 감소**"(또=OR, 서로 다른 동작). skills.csv `EffectDurationTurns=1`. DescKey `Skill.Block.Desc`="다음 1턴 **동안** 받는 피해를 50% 감소"(지속). §5-5 "막기 터틀 = 자기만 보호**(적1)**".
- **문제**: (a) *한 대 막고 소멸* vs (b) *본인 다음 턴까지 모든 피격 감소* 두 해석이 공존. 1:1(§5-3)은 사이 피격이 1회뿐이라 차이가 안 드러나지만, **실게임 4v4에선 다음 자기 턴까지 최대 3~4명이 집중 타격** → 막기 실효가 해석에 따라 최대 4배 진동. (b)면 집중공격이 무뎌져 피어/터틀이 강해지고 §5-5 "막기 터틀 불가" 판정의 전제가 무너진다. 추가로 **Duration 감소 시점**(누구 턴에 −1?)이 미정 — 매 유닛턴 감소면 (a)에 수렴, 본인턴만 감소면 (b)에 수렴.
- **재현**: 4v4, 전사가 t=k에 막기. 적 3명이 전사 다음 자기턴 전에 순차 타격. (a)면 1타만 반감·2·3타 풀피해, (b)면 3타 전부 반감. TTK/총턴 예산이 갈림.
- **확신도**: 모호성 CONFIRMED / 밸런스 영향 PLAUSIBLE(해석 확정 전 정량 불가).
- **필요조치**: 1줄 확정("소비=1회" 또는 "지속=본인 다음 턴 시작까지, 그 사이 모든 피격")을 §8·DescKey·§5-5에 일치시켜야.

### H2 — 사망 유닛 턴큐 처리 모순 (모순+명세공백) — Director 지적 축
- **근거(인용)**: §7-2 "사망 유닛은 … **제거/스킵** … 큐 자체 순서는 불변(**빈 슬롯만 스킵**)". §8-8 "… **턴큐제거 이벤트** …".
- **문제**: "제거"(배열 splice)와 "빈 슬롯만 스킵"(길이 고정·표식 스킵)은 **다른 구현**이다. 기존 턴제MVP식 `Current=(Current+1)%길이`에서 **배열을 실제 splice하면**: 현재 인덱스 **앞** 유닛이 죽어 좌측 시프트 → 다음 유닛 **스킵**, 또는 뒤가 죽어 **중복** 발생. 최악은 한쪽 전멸 순간 큐 길이 0 → **%0 크래시**(승패체크와 동프레임이라 회피될 수도 있으나 순서 미보장). 관대 해석("빈 슬롯 스킵"=길이 불변)이 안전하지만 §8-8 "제거"가 splice를 유도.
- **재현**: 큐 `[A1,B1,A2,B2]`, index=2(A2 행동)이 B1(index1) 처치. splice→`[A1,A2,B2]`(A2가 index1로 이동), 다음 `(2+1)%3=0=A1` → **B2 턴 스킵**.
- **확신도**: 문구 모순 CONFIRMED / 런타임 부정합·%0 PLAUSIBLE(naive splice 구현 시).
- **필요조치**: "**큐 길이 불변 — 사망=빈 슬롯 표식, 순회 시 스킵, splice 금지**" 1줄로 §8-8을 §7-2에 정렬. (verifier 실증 TC로 확인.)

### H3 — §8 판정순서에 쿨다운 세팅 단계 부재 (명세공백)
- **근거(인용)**: §8 판정순서 **1~9 어디에도 "시전 스킬 쿨다운 부여"가 없음**. 각주만 "쿨다운도 유닛별 (SkillId→남은쿨턴) 상태 필요". §0 "스킬 게이팅은 **쿨다운만**".
- **문제**: 알파 유일의 자원 게이팅이 쿨다운인데, 절차에 세팅 단계가 없어 **문자 그대로 구현 시 쿨 미적용** → 파이어볼(61~65)·베기·치유 매턴 연타 → §5 전 시뮬 무효, TTK 1~2턴으로 붕괴. 또 **쿨/막기 Duration 감소 시점**(본인 턴? 매 유닛턴?)이 절차에 없음(H1과 동근).
- **재현**: 마법사 t1 FB, 세팅 누락 시 t2 FB … 전사 90 → 2턴 사망.
- **확신도**: 절차 공백 CONFIRMED(유능한 엔지니어면 추론 가능 → 치명 아님, 그러나 계약 명세로는 공백).
- **필요조치**: §8에 "9.5 시전 스킬 CooldownTurns 부여" + "감소 시점=본인 턴 시작(권장)" 명문화.

### H4 — R1 완화책(막기/힐 반전)이 최적 선공 버스트엔 불성립 (밸런스+모순)
- **근거(인용)**: §5-3 "**막기로 반전**: 전사가 파이어볼 예상턴에 막기 → … 전사 생존 후 역전. 힐은 지속으로 개입." R1 "**막기/힐이 부분 완화**".
- **재현(독립 재계산)**: 1:1 전사(90/Def10/Atk40) vs 마법사(80/Def6/Atk42), **선공=마법사**(FB 오프너 최적). 후공 전사의 **모든** 대응 라인 전패:
  - 즉공(slash): M FB→29, W slash→마34, M basic→W **−3 패**.
  - 막기후공: M FB→29, W block, M basic(반감16)→13, W slash→마34, M basic→W **−19 패**.
  - 2연막기: 동일하게 **전패**.
  - **원인**: 고정 순환 큐라 후공 전사는 **오프너 파이어볼을 사전 막기 불가**(마법사가 먼저 행동). 막기에 턴을 쓰면 그만큼 딜을 못 넣어 오히려 더 뒤처짐. 전사 킷엔 **힐 없음** → "힐 개입"도 1:1 열세측엔 무의미.
- **문제**: "선공이 1:1을 결정"은 맞으나(R1), **막기/힐이 이를 반전한다는 서술은 상대의 비최적(파이어볼 미개시)·또는 전사가 애초 선공일 때만 성립**. §5-3의 "막기 반전" 행은 최적 플레이에서 도달 불가.
- **확신도**: CONFIRMED. 알파 vs-AI 진행 자체는 막지 않음(R1은 이미 알파 허용) — **문서의 완화 주장만 정정** 필요.

### H5 — 파이어볼 vs 막기 수치 오류 (모순, 게임영향 미미)
- **근거(인용)**: §5-1 "파이어볼 vs 막기(전사) | **28** (61→×0.5)", §5-3 "61이 **28**로".
- **재계산**: §8 순서 base=floor(42×1.7)=71, 71−10=61, ×0.5=30.5, floor→**30**. (어떤 순서 조합으로도 28 불가: 25/30만 가능.) → 확정공식대로면 **30**.
- **문제**: 시뮬표가 확정 공식과 불일치. 게임 영향은 없음(30·28 모두 전사 생존). 그러나 **§5 시뮬표가 최종 공식으로 재계산되지 않았을 신호** → QA/구현 대조 시 불일치. verifier가 표를 정답지로 쓰면 오탐.
- **확신도**: CONFIRMED.

## 3. 경(Low) 상세 (재현·근거 요약)

- **L6 타겟/AI 정책 공백**: §5-4 총턴23은 "최저 index 집중" 가정. 적 AI의 스킬선택(막기·힐 시점, 힐 대상)·타겟팅 규칙 미정의(알파 vs-AI 필수 입력).
