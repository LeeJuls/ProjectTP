## 1. 무엇이 됐나 — 스코프 & 확정 메커니즘

**스코프**: 피격 시 `"-{dmg}"` 플로팅 텍스트 표시.

**확정 메커니즘 = 스크린스페이스**([[야간큐_TC]] ③ "메커니즘 확정" 문단 그대로 구현):

- **`WBP_BattleHUD.Panel_FloatingNumbers`**: `CanvasPanel`, `HitTestInvisible` — 스폰된 숫자 위젯들의 부모.
- **`WBP_DamageNumber`**: `Txt_Amount`, 색 `#FF6B62`, 폰트는 Roboto 폴백(Jost 미보유).
- **`Anim_Float`**(2.4s, 4키×5트랙): `Opacity 0→1→1→0` · `TranslateY +10→0→−26→−36` · `Scale 0.85→1→1→0.95`.
- **`Manager.SpawnDamageNumber(Amount, WorldLoc)`**: `ProjectWorldToScreen` → `CreateWidget` → `"-"+ToString` concat(콤마 그룹핑 원천회피) → `AddChildToCanvas` → `SetPosition`.
- **위젯 자기소멸**: `Construct` → `PlayAnimation` → `Delay 2.4` → `RemoveFromParent`. ★애니메이션 에셋 길이가 5.0s로 보고되는 이슈([[언리얼_MCP_실전노하우]] §28 함정㊻) 때문에 `Finished` 이벤트는 쓰지 않고 `Delay` 고정값으로 수명을 제어.
- **HUD 생성 가드**: `InitBattle` 초입에 `IsValid` 가드 추가 — 재전투 시 HUD 중복 생성 방지.
- **`Menu_SkillMenu` 기본 `Collapsed`**: F7a가 스킬 메뉴 배선과 함께 가시성을 복원할 예정.

## 2. `ResolveHit` 콜사이트 — 3-way 병합점 스플라이스

`SpawnDamageNumber` 호출을 `ResolveHit`의 `IfThenElse_9` **앞**, 사망·생존·생존자없음 세 갈래가 전부 합류하는 **3-way 병합점**에 스플라이스했다. 이 위치 덕분에 **막타(킬링블로우)에도 데미지 숫자가 항상 표시**된다(DF-05 PASS의 구조적 근거). 노드 수 149→153, 컴파일 0에러.
