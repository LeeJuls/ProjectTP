# 야간F8 완료 — 광폭화 배율 결선 게이트 통과

> 대상: [[광폭화_재검증]] §7(F8 광폭화 공식 스펙 · TC-Z01~05) · [[스탯_전투공식_v1]] §8(판정순서 step2 HEAL·step4 base) · [[야간큐_TC]](같은 야간 세션 TC 계보)
> `BerserkMult(1+0.05×max(0,turn−30))`를 `ResolveHit` step4의 곱 최종 항으로 결선해, F1에서 손계산으로 승인된 광폭화(anti-stall) 배율 스펙을 처음으로 그래프에 실장하고 PIE 실측 4게이트(무회귀·turn40/50 스케일·힐불변)로 검증까지 마친 결과를 기록한 문서. 같은 야간 세션에서 F6·F7a에 이어진 게이트.

## 1. 무엇이 됐나 — 스코프 & 구현 체인

**스코프**: `BerserkMult(turn) = 1 + 0.05 × max(0, turn − 30)`를 `ResolveHit` step4의 곱 최종 항으로 추가. **단일 floor 유지**(중간 floor 신설 없음) · **곱 순서 `Atk × OutgoingAtkMult × PowerRate × BerserkMult` = M2로 이미 확정된 순서를 그대로 준수**(변경 없음).

**구현 체인**(신규):
1. `GetTurnCounter` → `−30` → `Max(0)` — 전부 **정수 도메인**에서 완결(`TurnCounter`가 int라 뺄셈·클램프 모두 int 연산).
2. → `ToFloat` — **★int 연산이 끝난 뒤에만** float로 전환한다. [[야간큐_TC]] F6-05가 확립한 원칙("정수 연산과 float 변환의 순서를 정확히 지켜야 함정을 피한다")의 재적용 — F6-05는 "나눗셈 앞에 ToFloat"였고 이번은 "뺄셈·클램프 뒤에 ToFloat"로 방향은 반대지만, **연산이 요구하는 숫자 도메인이 바뀌는 경계에서만 변환한다**는 원칙은 같다.
3. → `×0.05` → `+1.0` — `BerserkMult(turn)` 산출 완료.
4. → 기존 `Pow9`(step4의 기존 `Atk×OutMult×PR` 곱 결과 노드) 출력과 곱.
5. → `Floor`의 `A` 입력 핀에 **단일 소스**로 직결 — 핀 원문 조회로 배선을 직접 확인(추정이 아니라 조회값 근거).

**floor 후보 역추적 분리**: 그래프에서 Floor 후보가 3개 검색됐다. 이 중 HEAL 전용 Floor(`CallFunction_45`)와 DAMAGE step4 Floor(`CallFunction_50`)를 역추적으로 분리 식별해, 신규 BerserkMult 체인이 `CallFunction_50`(DAMAGE)에만 연결되고 `CallFunction_45`(HEAL)는 무접촉임을 교차 확인했다 — [[광폭화_재검증]] §7이 요구한 가드레일("step2 HEAL은 미접촉 → Atk 스탯 불변 → 힐 배율 안 붙음")을 배선 수준에서 실증한 것.

## 2. 스캐폴드

`TurnCounter` = **Instance Editable 플래그**로 켜둠(verifier가 30턴+ 시나리오를 턴 가속으로 도달시키기 위한 MOCK 경로) — **★F9a 스캐폴드 정리 시 원복 대상**으로 명기해둔다. 이전 야간 게이트가 `BP_BattleSpawnPoint.Hp`에 같은 목적으로 남긴 선례와 동일 패턴(용도 종료 후 반드시 원복).

## 3. 검증 — PIE 실측 4게이트

| 게이트 | 조건 → 기대 | 로그 원문 |
|---|---|---|
| turn1(무회귀) | turn≤30 → Mult=1.00, 전사 basic 데미지 기존값 불변 | `BattleLog\|turn=1\|...\|dmg=30\|hp=60` |
| turn40 | Mult=1.50 → 전사 basic 50 | `BattleLog\|turn=40\|...\|dmg=50\|hp=10` |
| turn50 | Mult=2.00 → 전사 basic 70 | `BattleLog\|turn=50\|...\|dmg=70\|hp=20` |
| turn52(힐 불변) | HEAL 경로 무접촉 → 치유 -33 고정 | `BattleLog\|turn=52\|...\|dmg=-33\|hp=90` |

4건 전부 [[광폭화_재검증]] §7 기대값 표(turn40=50·turn50=70·힐 33 고정 행)와 정확히 일치. **에러 0**(런타임 크래시·None 참조 없음).

**이번 세션 커버리지**: [[광폭화_재검증]] §7의 TC-Z01(turn≤30 무효과 표본)·Z02(turn40/50 스케일)·Z03(힐 미스케일)을 게이트했다. turn31 +5% 표본(Z01 잔여)·Z04(양측 대칭)·Z05(종결 보장), 그리고 `plan.md` TC-F8-02·06·07·08(파이어볼 스케일·대칭·종결·막기 미스케일)은 이번 세션에서 미실측 — 후속 세션(F9a 풀회귀)으로 이월.

## 4. 실측 메모 — 파티 스탯 비균일 재확인

F0-① 배정표 그대로 재확인: **A1·A2 = 전사 Atk40/Def10**, **A3 = 마법사계 Atk42/Def6**, **A4 = 치유 담당**(전2+마2 미러 배정 불변). 전사 데미지 검증은 **A1/A2 공격자 + B1(Def10) 타겟 고정**이 안전 — B2~B4는 스프라이트가 시각적으로 구분되지 않아 오타격(엉뚱한 유닛을 잘못 관측·오판정) 위험이 있다.
