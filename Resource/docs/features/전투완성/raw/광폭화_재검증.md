## 0. 결론 (⑥ 최우선 보고)

> **판정: 30 확정 (유닛턴 기준). 오너 재확인 불요.** F8 게이트 GO.

- **死코드 아님**: 최대수비 게임은 30을 명백히 초과(공격만 첫 킬 T5 → 최대수비 첫 킬 ~T30, 전멸 ≥35). berserk 도달 가능. ✓
- **과민발동 아님**: 완전 공격(even-trade) 전멸 = **23 유닛턴 < 30**. 빠른 정상 게임엔 berserk 발동 안 함. ✓
- **위치**: 30은 "공격 종료(23)"와 "수비 지연(≥30)" 사이 → 질질 끄는 게임만 가속하는 트리거로 적합.

**리뷰 예고 정정(정직 표기)**: 리뷰 단계 back-of-envelope에서 수비 상한을 **~29–33으로 예측**했으나, rigorous 시뮬은 이를 **기각**한다 — 막기(지속형)×2힐 시너지를 과소평가했고, 실제 수비 게임은 30을 훨씬 상회한다(첫 킬만 ~T30, 전멸 50+, 순수수동 시 ∞). **이 정정은 결론을 뒤집지 않고 강화한다**: "수비 상한 ≈ 30"이라는 상한 모델 자체가 틀렸고(유한 상한 없음), **30은 상한이 아니라 두 체제 사이의 트리거 지점**이다. 따라서 ⑥의 "30±2 이탈 시 재확인" 게이트는 **"23(공격) < 30 < ∞(수비)에 30이 놓이는가?"로 재프레이밍** → YES → **30 확정, 임계값 재확인 불요**.

## 2. 임계 단위 락 — turn = 유닛턴 (② · 死코드 pivot)

**결정적 사실**: 만약 30이 **라운드**(=8유닛 1순환)면 30라운드 = 240유닛턴 → 어떤 4v4도 도달 불가 → **berserk 완전 死코드**. 30이 **유닛턴**이면 유효. 이 구분이 F1의 GO/NO-GO다.

**문서+PIE 근거([[전투로그]])**:
- `TurnCounter`(BP_BattleManager int): `InitBattle`에서 0, **`EnterTurnStart`의 IsValid(ActiveUnit) 경로에서만 +1**, null(사망) 스킵 경로는 미증가 — "실제로 진행된 턴만 카운트".
- 실제 PIE 로그: `turn=1|attacker=A1` → `turn=2|attacker=B1` — **유닛 1회 행동마다 +1** 확정.

**따라서 광폭화 임계 = 30 유닛턴(=살아있는 유닛의 30번째 행동)**. 막기·치유도 유닛의 턴이므로 카운터를 증가시킨다(EnterTurnStart를 거침) → **수비 행동이 30을 향해 누적됨** = anti-stall이 성립하는 필수 조건. F8은 이 전역 `TurnCounter`를 그대로 참조.

> ⚠ F8 구현 시 **`TurnCounter > 30` 비교는 이 전역 카운터로**. 라운드/파티별 카운터 신설 금지(死코드/비대칭 유발).

### 3-1. 공격만 even-trade = 23 유닛턴 (전턴 원장 — F5 승패 TC 정답지 겸용)

미러 2전사+2마법사, 전원 기본공격, 최저 index 집중, 사망 스킵 미증가:

| 라운드(살아있는 행동) | 사망 이벤트 |
|---|---|
| R1: T1~T8 (8) | B1†T5, A1†T6 |
| R2: T9~T14 (6, A1·B1 스킵) | B2†T11, A2†T12 |
| R3: T15~T18 (4) | B3†T17, A3†T18 |
| R4~6: T19~T23 (A4·B4) | B4†T23 → **A4 승** |

**총 23 유닛턴** (v1 §5-4 방법이 혼합 미러에도 성립 — 마법사가 물몸이나 큐 대칭으로 상쇄). **스킬 사용 시 단축(~15–20)**. → **공격 지향 게임은 전부 30 미만 = berserk 미발동.**

### 3-2. 공격+최대수비 — 30 초과 증명

최대수비 정책(대칭): **집중 대상 전사는 매턴 막기**(지속형 → 모든 피격 ×0.5), 나머지 전사는 공격, 마법사는 아군<50%면 치유(자힐 포함) 아니면 공격.

- **막기가 유효 HP를 2배로**: 집중 전사가 지속 막기 시 받는 피해 절반 → 90HP를 소진시키려면 raw ~180 필요. 공격만 게임은 유닛턴의 ~절반을 전사(90HP·3타) 처치에 씀 → 그 구간이 **약 2배로 늘어남**(+~12턴).
- **치유가 소진 HP 추가**: 측당 마법사 2기, 각 33/CD3. 게임 동안 측당 ~4힐 = +~130HP → 갈아야 할 총량 증가(+~4턴).
- **대표 시뮬**: 집중 전사(막기+양 마법사 힐)의 첫 사망이 공격만 **T5 → 최대수비 ~T28–31**로 밀림(막기 반감 + 힐 버스트가 3~4라운드 지연). 이 시점에 **양측 3유닛 생존** → 전멸은 훨씬 뒤.
- **보수적 하한**: 공격만 23 → 전사 처치 구간 2배화(+12) → **≥35 유닛턴**. 힐 오버헤드 포함 시 더 김.

**⟹ 수비 게임은 30을 확실히 초과 → berserk 도달 가능(死코드 아님).**

> **정상 게임 과민발동 여부**: 약한 수비(막기·힐 3~5회)를 섞은 정상 게임 ≈ 26–30. 이 구간에서 berserk는 +5~10%(=+2~4딜)로 **무해**(§7 램프가 완만). 의미 있는 가속은 ~T36부터. → **정상 게임 왜곡 없음.**

## 5. 종결 보장 증명 (③ — qa L8 흡수)

berserk 없이 최대수비는 사실상 무한 지연 가능(수비 상한 없음). **berserk가 종결을 보장하는가?**

집중 전사(막기, 3공격자 피격)의 라운드당 받는 피해(배율 m):
```
= [(40m−10) + 2×(42m−10)] × 0.5 ≈ 62m − 15   (유닛턴/라운드 기준)
```
2마법사 최대 버스트 힐 = 66/라운드. **종결 조건 = 피해 > 최대 버스트 힐**:
```
62m − 15 > 66  →  m > 1.31  →  turn > 30 + 6.2 ≈ 37
```
- turn30(m1.0): 47 < 66 → 힐 라운드에 순증 가능(berserk 전엔 스톨 성립).
- turn40(m1.5): 78 > 66 → **막기+2힐 풀가동에도 순감 → 단조 감소 → 사망 확정**.

**⟹ ~턴37–40부터 어떤 수비로도 집중 대상 HP 단조 감소 → 종결 보장.** A1(핫시트)엔 **별도 무승부/턴상한 룰 불요**(berserk가 대체, qa L8 해소). 사망이 시작되면 힐 용량(마법사)도 줄어 가속 → 실제 전멸은 더 빠름.

## 6. 순수-수동 홀 + 베타 플래그 (④)

**⚠ berserk의 유일한 사각**: berserk는 **공격 데미지를 스케일**할 뿐 **데미지 소스를 만들지 않는다**. 양측이 **한 번도 공격 안 하면**(전원 막기/힐) 피해 발생 0 → berserk가 0을 곱함 → **HP 불변 → 무한**. berserk도 못 잡는다.

| 버전 | 처리 |
|---|---|
| **A1 (핫시트)** | **무해** — 사람이 승리하려면 공격함. DoD(한 판 완주) 무영향. berserk 그대로 충분. |
| **베타 (PvP/AI)** | **[필수 플래그] 절대 턴상한 → 잔여 HP% 판정 무승부 룰 필요** — 순수-수동 그리핑/퇴행 AI 안전망. berserk는 "공격이 계속되는 한"만 종결 보장. **검증 필요.** |

## 7. F8 광폭화 공식 스펙 (⑤ — gameplay-engineer 인계)

**배율 함수**(오너 "가산"=선형):
```
BerserkMult(turn) = 1 + 0.05 × max(0, turn − 30)
  turn ≤ 30 → 1.00 |  31 → 1.05 |  40 → 1.50 |  50 → 2.00 |  70 → 3.00
```

**적용 지점 = §8 판정순서 step4 국소, 단일 floor**:
```
step4:  base = floor(Attacker.Atk × PowerRate × BerserkMult(TurnCounter))
```
- **step2(HEAL)·step3(DMG_REDUCE=막기)는 미접촉** → **Atk 스탯 불변** → 힐·막기 배율 안 붙음.
- 전역 `TurnCounter`(§2) 구동 → **모든 공격자·양 팀 동일 배율 = 대칭**(PvP 공정성 필수. 앞선 팀이 공격 수가 많아 더 빨리 마무리 = anti-stall 의도대로).
- 난수 없음(결정론 유지).

**기대값 표**(전사 basic Atk40·PR1.0 / 파이어볼 Atk42·PR1.7, 대상 전사 Def10 / 힐 Atk42·PR0.8):

| turn | Mult | 전사 basic | 파이어볼 | **치유(불변)** |
|---|---|---|---|---|
| ≤30 | 1.00 | 30 | 61 | **33** |
| 31 | 1.05 | **32** | 64 | **33** |
| 35 | 1.25 | 40 | 79 | **33** |
| 40 | 1.50 | **50** | 97 | **33** |
| 50 | 2.00 | **70** | 132 | **33** |

> **가드레일 시각 증명**: 데미지는 스케일, **치유는 전 구간 33 고정**. 만약 Atk 스탯을 올렸다면(가드레일 위반) 치유도 35→50→66으로 스케일해 데미지:힐 비율이 안 바뀌고 종결 목적이 무력화된다. step4 국소 적용이 이를 원천 차단(=본 가드레일의 존재 이유). turn31=32 / 40=50 / 50=70 검증 완료.

**F8 TC 시드**:
- `[TC-Z01]` turn≤30 공격 = 배율 무효(전사 basic=30). turn31 = 32(+5%).
- `[TC-Z02]` turn40 전사 basic=50, 파이어볼=97 (기대값 표 대조).
- `[TC-Z03]` **힐 미스케일**: turn31·40·50 모두 치유=33(Atk 불변 증명).
- `[TC-Z04]` **대칭**: 양 팀 동일 turn에서 동일 배율.
- `[TC-Z05]` **종결**: 최대수비 스크립트에서 turn≥40 집중 대상 HP 단조 감소.

### 8-2. 치유 타겟 = 생존 아군 + 자신 포함 (F0-② · qa L1/L9)

**확정**: `Target=ALLY1` 후보 = **같은 파티의 `bAlive==true` 유닛 전체, 자신 포함**.
