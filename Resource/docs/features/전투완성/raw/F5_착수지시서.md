## ★BLOCKER 판정 5건 (착수 전 필독)

### B1 — `bBattleOver` 카운트 대상 정정 ★plan.md의 논리 오류

**[[plan]] L382 원문이 틀렸다**:
> ~~"상대팀(Target과 **반대** bIsParty) 생존수 카운트"~~

**왜 틀렸나**: Target(죽은 유닛)의 **반대** 팀 = **공격자 팀**이다. 공격자는 방금 행동했으므로 **항상 생존**해 있다 → 카운트가 절대 0이 안 됨 → **`bBattleOver`가 영원히 안 켜진다** → 적을 전멸시켜도 전투가 계속되고, 다음 턴 `ResolveTargetPool("ENEMY1")`이 빈 배열을 반환해 **소프트락**.

**✅ 확정**:
```
death 분기(ResolveHit step8) 안에서:
  survivors = TurnQueue 중 (bIsParty == Target.bIsParty) AND (bAlive == true) 의 개수
              ★ Target과 "같은 팀" — 죽은 쪽 팀이 전멸했는지를 본다
  if survivors == 0:
      bBattleOver = true
      WinningTeam = Attacker.bIsParty      (= Target.bIsParty 의 반대)
```

### B2 — `WinningTeam` 의미 통일 = **승자**

§5-2는 "승자", §5-4의 `BattleFinished` payload는 "전멸팀"이라 **정반대**였다.
**✅ 확정: 둘 다 `WinningTeam`(= 승자 = `Attacker.bIsParty`)로 통일.** `BattleFinished(WinningTeam)` 직결.
(이름이 `WinningTeam`인 이상 승자가 자연스럽고, A6에서 필요한 정보도 "누가 이겼나"다. §5-4 문구는 이 판정으로 정정된 것으로 간주.)

### B3 — `InitBattle` 완전 리셋 (a안 채택)

**문제**: 리셋 목록에 **Hp·bFreeze·RowIndex·ClickBox·bBlockActive·HP바** 등 7항목이 빠져 있었다. 특히 **`Hp`는 `BeginPlay`에서만 로드**되므로(F3), 재시작하면 전원 **"Hp=0 · 누운 채 · 클릭 불가"** 로 부활한다. 크래시가 안 나서 TC는 PASS인데 **실제로는 플레이 불가**.

**✅ 확정: 유닛에 `ResetForBattle()` 함수 신설** → `InitBattle`이 `ForEachLoop(TurnQueue)`로 전원 호출.

```
ResetForBattle()  [BP_BattleSpawnPoint]
  1. DT_JobStats(JobId) 재조회 → Hp = MaxHp = row.hp,  Atk = row.atk,  Def = row.def
     (또는 MaxHp가 이미 있으면 Hp = MaxHp 로 충분 — 스탯 재로드가 더 안전)
  2. bAlive = true
  3. ActiveStatuses = 빈 배열
  4. SkillCooldowns = 빈 맵
  5. bBlockActive = false,  BlockValue = 0
  6. SpriteMID: bFreeze = 0,  RowIndex = 0(IDLE),  FrameCount = 6,  TimeOffset = 0
  7. ClickBox 콜리전 복원 (QueryOnly)
  8. CachedUnitFrame.SetHp(Hp, MaxHp)   ← 화면 HP 바 복원
```
`InitBattle` 추가분: 전 유닛 `ResetForBattle()` + `SelectedTargets` Clear + `bBattleOver = false` + `WinningTeam` 초기화 + `BattleState = 0`

### B4 — 쿨다운 스윕·막기 해제 = **F5에서 구현** (3문서 3주장 해소)

`상태이상_확정`은 "기존 불변", `plan §F7-4`는 "F7 소관", Director는 "F5"라고 서로 다르게 말했다. **실제로는 F4에 접근자(`Get/SetSkillCooldown`)만 있고 스윕은 없다.**

**✅ 확정: F5에서 구현.** 이유: F5 설계의 핵심 근거가 **"TS3가 TS4보다 앞"**인데, TS3가 존재하지 않으면 그 순서 자체가 검증 불가능한 공허한 문장이 된다. TC-SE06/N06/N07도 실행 불가가 된다.

```
TS3 (EnterTurnStart 안):
  - ActiveUnit.SkillCooldowns 전 엔트리 −1 (하한 0)   ※ Key가 String인 Map
  - ActiveUnit.bBlockActive = false  (막기 해제)
```

### B5 — `bFreeze=1`은 DYING 재생 **후에**

**문제**: DYING과 동시에 `bFreeze=1`을 켜면 **즉시 마지막 프레임으로 점프** → **쓰러지는 동작이 화면에 안 나온다.** 오너의 필수 확인 항목이 "**쓰러져** 고정"인데 쓰러지는 과정이 사라진다.

**✅ 확정 순서** (`PlayHurtReaction`의 death 분기):
```
1. SpriteMID: TimeOffset = GetGameTimeInSeconds
2. SpriteMID: FrameCount = 5      (DYING = Row13, FrameCount 5)
3. SpriteMID: RowIndex   = 13
4. RetriggerableDelay(0.575)      (= 5/8 − 0.05, 재생 완료 대기)
5. SpriteMID: bFreeze = 1         ← 여기서 고정
   ★ idle 복귀 노드로 가지 않는다(기존 HURT 분기의 0.45초 후 idle 복귀와 분리)
```
**동반 함정**: `SetScalarParameterValue` 호출 시 **`declaring_class`를 MID로 명시하지 않으면 무음 실패**(노하우 §L130) → 시체가 계속 순환 재생된다. 반드시 명시할 것.

## HIGH 3건 (구현 시 반드시 반영)

### H-1 — TS1은 기존 노드 **앞에** 스플라이스 ★F9a 원장이 걸려 있다

현행 `EnterTurnStart`: `ActiveUnit 세팅 → IsValid → **SetTurnCounter(+1)** → MarkerOn → EnterAwaitCommand`

**TS1(사망 스킵)을 `SetTurnCounter` 뒤에 붙이면 사망 유닛도 TurnCounter를 +1 시킨다.** 그러면:
- 광폭화_재검증 §2(턴=유닛턴 정의) 위반
- **F9a 원장(23 유닛턴)이 통째로 붕괴** — 그런데 이 버그는 **F8/F9에서야 터진다**(늦게 발견 = 최악)

**✅ TS1을 `SetTurnCounter` 앞에 넣어라.**
```
EnterTurnStart:
  ActiveUnit = TurnQueue[CurrentIndex]
  TS1. Branch(ActiveUnit.bAlive == false) → True: [마커·카메라 없이] EnterTurnEnd  ★여기서 종단
  TS2. SetTurnCounter(+1)        (기존 노드 — 위치 불변)
  TS3. 쿨다운 스윕 −1 + 막기 해제  (B4)
  TS4. bSkipTurn = ActiveUnit.ActiveStatuses에 (STUN ∧ remainingTurns>0) 존재  ※차감 없이 판정만
  TS5. Branch(bSkipTurn) → True: StatusLog SKIP_TURN → EnterTurnEnd  ★여기서 종단
  TS6. MarkerOn → EnterAwaitCommand  (기존)
```
**검출 TC**: B1 사망 후 A1의 연속 두 턴 사이 `turn` 증분 == **7** (8이면 위반 — 사망자가 카운트됨)

### H-2 — 스킵 경로는 **exec 종단** 필수

TS1/TS5가 `EnterTurnEnd`를 호출한 뒤 **exec를 이어붙이면 안 된다**. 이어붙이면:
- 시체에게 `AwaitCommand`가 열리고
- `CurrentIndex`가 **2칸 튀어** 다음 유닛이 턴을 잃는다

**✅ `EnterTurnEnd` 호출 노드의 `then` 출력을 아무데도 연결하지 마라**(종단).

### H-5 — `bBattleOver`는 **`ResolveHit` 안에서만** 세팅

레이스 마진(0.95s)은 `then_0`/`then_1`이 **같은 Sequence**에서 갈라지기 때문에 구조적으로 보장된다.
**깨지는 유일한 경로**: 플래그 세팅을 `ResolveHit`(동기 Function) **밖**, 즉 `PlayHurtReaction`(latent 보유)의 Delay **뒤**로 옮기는 것 → 마진이 **0.37s로 축소**되어 **F6에서 역전**한다.

**✅ 규칙**: `bBattleOver`/`WinningTeam` 세팅은 **`ResolveHit` step8 death 분기 안에서만**. `PlayHurtReaction`은 **연출 전용**(플래그 세팅 금지).

## 구현 목록 (확정)

### BP_BattleManager
1. **신규 변수**: `bBattleOver`(bool) · `WinningTeam`(bool = 승자의 bIsParty)
2. **`BattleState = 6`(End)** 추가 (기존 Byte 매직넘버 체계 유지: 0초기/1TurnStart/2AwaitCommand/3AwaitTarget/4Executing/5TurnEnd/**6End**)
3. **`ResolveHit` step8 death 분기 확장**: `Target.bAlive=false` 직후 → **B1의 생존수 카운트** → 0이면 `bBattleOver=true` + `WinningTeam=Attacker.bIsParty`
4. **`EnterTurnStart` = TS1~TS6** (H-1 순서 엄수, H-2 종단)
5. **`EnterTurnEnd` 확장**:
   ```
   ① MarkerOff(ActiveUnit)   (기존, 카메라 복귀 포함)
   ①.5 TickStatusesAtTurnEnd(ActiveUnit)  ← 신규: ActiveStatuses 전 엔트리 remainingTurns −1,
        ≤0 제거 + StatusLog EXPIRE.  ★무분기(항상 실행) · ActiveUnit 한정 · ② 이전
   ② CurrentIndex = (CurrentIndex + 1) % TurnQueue.Length   (기존)
   ③ Branch(bBattleOver)  ← 신규
        True  → EnterEnd()
        False → Delay(0.35) → EnterTurnStart()   (기존)
   ```
6. **`EnterEnd()` 신규**: `BattleState=6` · `bInputLocked=true` · TurnMarker 전원 OFF(ForEach) · 하이라이트 전원 OFF · **`BattleFinished` 디스패처 방출(payload=`WinningTeam`)** · **teardown 금지**(비파괴 — `InitBattle` 재호출로 재시작 가능해야 함)
7. **`InitBattle` 리셋 확장** (B3): 전 유닛 `ResetForBattle()` + `SelectedTargets` Clear + `bBattleOver=false` + `WinningTeam` 초기화
8. **`TickStatusesAtTurnEnd(Unit)` 신규 함수**: 만료 제거는 **임시배열에 담아 통째 재대입**(인덱스 시프트 버그 회피)

### BP_BattleSpawnPoint
9. **`PlayHurtReaction`에 death 분기 추가**:
   ```
   Branch(bAlive)
     True(생존)  → 기존 HURT 경로 그대로 (Row12/FC4 → 0.45s → idle 복귀)
     False(사망) → DYING 경로 (B5 순서 엄수: Row13/FC5 → Delay 0.575 → bFreeze=1, idle 복귀 없음)
   ```
10. **`ResetForBattle()` 신규 함수** (B3 스펙 그대로)
11. **사망 시 `ClickBox` → NoCollision** (타겟팅 제외. `ResolveTargetPool`이 이미 `bAlive` 필터를 하므로 이건 이중 안전장치)

## 검증 (게이트)

**기능 TC**(verifier — 이게 통과해야 F6 착수):
- **TC-F5-01★**: 킬링블로우 레이스 — 마지막 생존자 사망 시 최종 상태가 **End**로 유지(then_1의 EnterTurnEnd가 TurnStart로 되돌리지 않음). **2회 재현**
- **B1 검증**: 한쪽 팀 전멸 → `bBattleOver=true` + `EnterEnd` 도달 (**이게 안 되면 소프트락**)
- **H-1 검증**: 사망자 발생 후 한 바퀴의 `turn` 증분 == **7**(8이면 FAIL)
- **H-2 검증**: 사망 유닛 턴에 `AwaitCommand` 미진입 + `CurrentIndex` 1칸만 이동
- 큐 길이 불변(8, splice 금지) · 양측 대칭 종료 · `InitBattle` 재시작 후 **정상 플레이 가능**(Hp 복원·기립·클릭 가능)
- 쿨다운 스윕(TS3) 동작 · STUN 1턴 스킵(δ 검산) · ATK_DOWN 2턴

**비주얼 TC**(오너 육안 — 진행 안 막음):
- **쓰러지는 동작이 보이고** 마지막 프레임에 고정(꿈틀거림 없음) ← B5가 이걸 지킨다
- 시체 누적 구도 · 한 판이 끝나는 느낌
