> 23유닛턴 기본공격 even-trade 시나리오를 클릭 자동화로 완주해, ③~F8 야간 게이트가 라이브 상태에서도 F1/v1이 확정한 정답지 원장과 diff 0임을 실증하고, ⑤가 이월했던 died 필드 반영 기대원장을 확정본으로 갱신한 결과를 기록한 문서. 야간 세션 게이트(③③b④⑤F6F7aF8) 중 마지막.

## 1. 무엇이 됐나 — 스코프 & 방법론

**스코프**: [[광폭화_재검증]] §3-1이 정의한 "공격만 even-trade" 23유닛턴 시나리오(전원 기본공격, 최저 index 집중, 사망 스킵 미증가)를 라이브 빌드로 완주. turn≤23이라 전 구간 `BerserkMult=1.00`(F8 미접촉) · 스킬메뉴 미경유(F7a 미접촉) — **순수 코어 루프(턴진행·데미지·사망·로그) 회귀 검증**이 목적이다. F7a 5스킬·F8 광폭화 자체 검증은 각자의 완료 문서([[야간F7a_스킬메뉴_완료]]·[[야간F8_광폭화_완료]])가 이미 커버했고, 본 게이트가 재검하지 않는다.

**방법론**: 매 턴 `activeUnit`을 먼저 조회해 기대 공격자와 일치하는지 선확인한 뒤 클릭을 실행하는 방식으로 23턴 전량을 클릭 자동화 — 오타격·순서이탈을 턴 단위로 원천 차단했다.

## 3. 사망 스킵 실전 — H-1 유효성 재확인

23턴 진행 중 `SKIP_TURN_DEATH`(TS1, [[야간⑤_로그보강_완료]] 보강 이벤트)가 다수 방출됐고, 그때마다 `TurnCounter`가 **불변**(사망 유닛의 턴이 카운터를 올리지 않음)임을 확인했다 — [[광폭화_재검증]] §2 H-1("사망 스킵은 미증가")이 라이브 23턴 전량에서 유효함을 재확인한 것이다. `extract_battle_log.py` 파서도 이번 세션 로그에서 **255줄 정상 추출**로 호환성을 재확인했다([[야간⑤_로그보강_완료]] LOG-06 파서 회귀의 연장 재검).

## 4. 회귀 스윕 — 걸음·카메라·재전투 루프

- **걸음/카메라 24회**: 유닛 이동(WalkForward)·액션캠 컷 24인스턴스 전부 정상(F6 F6-07 무접촉 원칙과 정합, 오작동·정지 0).
- **End→RESTART→재전투**: End 화면 도달 → 클릭 → `InitBattle`(RESTART) → 재전투 T1이 **원 T1과 동일값**(`A1→B1|dmg=30|hp=60`) — [[야간④_End버튼_완료]] `ResetForBattle`/`InitBattle` 리셋 경로가 이 원장 기준에서도 재현 가능함을 확인했다.
- **에러 0**: 스윕 전 구간 런타임 에러 0([[언리얼_MCP_실전노하우]] §28 함정㊾ 한국어 로케일 패턴 "None에 액세스" 스캔 기준).
- **데미지폰트 렌더 = 오너 육안 보류**: 이번 게이트는 BattleLog `dmg`와 원장 값의 **데이터소스 정합**만 기계로 확증했다. 화면상 폰트 룩·타이밍 체감은 [[야간③b_데미지폰트_위치수정_완료]] §6 이월 그대로 오너 육안 대기.

## 5. 정정 노트 — 사망 라인 개수

[[야간큐_TC]]의 LOG-04 항목이 표기한 "F9a 기대원장 사망 라인 8개"는 **오탈자**다. 본 문서 §2 원장 기준 사망 라인은 **정확히 7개**(T5·T6·T11·T12·T17·T18·T23 — B1·A1·B2·A2·B3·A3·B4). [[야간큐_TC]]가 나열했던 명시적 목록(B1†T5·A1†T6·B2†T11·A2†T12·B3†T17·A3†T18·B4†T23) 자체도 7항목이라 "8개"는 숫자 표기만의 오류였음이 이번 완주로 재확인된다. [[야간큐_TC]]는 본 문서와 함께 7개로 정정한다.

## 6. 스캐폴드 처리 판단 (Director)

`BP_BattleSpawnPoint.Hp`([[야간④_End버튼_완료]] §6)·`TurnCounter`([[야간F8_광폭화_완료]] §2)의 **Instance Editable 플래그 원복을 F7b 마감 시점으로 연기**한다. 원래는 F9a(본 게이트) 시점 원복 대상으로 명기돼 있었으나, 아침 오너 육안 확인과 F7b 검증 과정에서 동일 MOCK 경로(턴 가속·Hp 강제조작)를 재사용할 가치가 있어 존치가 더 낫다는 판단이다. 원복 대상이라는 표기 자체는 각 단계 원 기록(④·F8)에 그대로 유지 — 이번 결정은 "언제 원복하느냐"만 바꾼다.

## 7. 잔재 — orphan 노드

[[야간③_데미지폰트_완료]] §4의 `SpawnDamageNumber` 내 harmless orphan 노드 1개(연결 0)를 포함해, 야간 세션 동안 문서화된 orphan 노드들은 이번 풀회귀에서도 기능·수치에 영향이 없음을 재확인했다 — [[F5-2_완료]] §3 선례 그대로 정리하지 않고 존치한다.
