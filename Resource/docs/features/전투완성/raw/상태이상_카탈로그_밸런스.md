## 2. D1 — 알파 카탈로그 범위

- **D1-안A (권장): STUN + ATK_DOWN 2종** — 오너 예시 그대로.
  - 이 2종이 구현 아키타입 2개를 정확히 1개씩 커버: **턴 박탈형**(STUN — 기존 사망 턴스킵 패턴 변형)과 **판정식 배율형**(ATK_DOWN — 광폭화 step4 배율 패턴 재사용). DEF_DOWN/ATK_UP/도트 등은 전부 이 둘의 변주라, 아키텍처가 한번 서면 **베타에서 데이터 행+분기 1개 추가**로 끝난다.
  - A1 DoD는 "스킬 5종 완주"다. 붙일 자리가 현실적으로 2곳(베기·파이어볼, §5)뿐이므로 3종째부터는 **발동 스킬이 없는 死데이터** — F9에서 검증 불가, qa 지적 예상.
- D1-안B: 4종(+DEF_DOWN, ATK_UP) — 카탈로그를 먼저 채워두는 안. 장점: 베타 선행 설계. 단점: 死데이터 2행, 오너 enum 어휘만 늘고 검증 경로 없음, 상향 버프(ATK_UP)는 자기강화 스택 점검이 별도로 필요(과설계).
- **권장 = 안A.** 단 카탈로그 표(§3)는 확장 열(위상/틱 규칙)을 갖춰 베타 추가가 행 삽입으로 끝나게 설계.

### 3-1. STUN = 자기 턴 1회 스킵 — 근거
- **패턴**: 기존 사망 턴스킵(EnterTurnStart에서 행동 생략)과 동일 지점의 변형. **차이 = 시한부**: 사망은 영구 스킵(`bAlive=false`, 알파에 부활 없음), 기절은 지속 소진 후 **자동 회복**. 따라서 `bAlive` 재활용 금지 — 별도 상태(예: StunRemaining)가 필수(사망 애니·타겟팅 제외·승패 카운트가 전부 bAlive에 묶여 있어 오염됨). 기절자는 사망자와 달리 **타겟팅·치유 가능**해야 한다.
- **지속 1턴인 이유(액션 이코노미)**: even-trade 원장(23 유닛턴, [[광폭화_재검증]] §3-1)에서 유닛당 평균 행동 = 23/8 ≈ **2.9회/판**. 행동 1회 박탈 = 해당 유닛 판 기여의 **~34% 삭제**(박탈 가치 ≈ 평균 공격 1회 ≈ 원시딜 30~46 + 힐/막기 계획까지 봉쇄하는 템포). 2턴이면 유닛 반절 삭제 = 프록 하나가 판을 결정 → 확률 이벤트가 감당할 크기가 아님. **1턴 고정.**
- **파생 계약**(밸런스가 요구하는 불변식, 배선은 구조 소관):
  - 기절 턴에도 **쿨다운은 −1 진행**(행동만 박탈, 시계는 흐름 — 이중 처벌 방지).
  - 기절과 무관하게 자기 턴 "시작"은 발생 → **막기 버프는 기존 계약대로 그 시점에 해제**(기절이 막기를 연장하지 않음).
  - D8: 기절 스킵 턴의 TurnCounter는 **+1 권장**(생존 유닛의 소모된 턴. 사망 스킵 미증가와 구분. 기절 다용 게임도 광폭화 anti-stall에 포섭됨) — 단 카운터 오너는 구조 쪽이므로 **[구조 확인 필요]**.

### 3-2. ATK_DOWN = Atk ×0.75, 자기 턴 2회 — 근거
- **감산 Def의 증폭 효과**: Atk 25% 감소가 최종딜에서는 **약 28~34% 감소**로 나타난다(빼는 Def는 그대로라서). 약화 매트릭스(단일 floor, 곱 순서 Atk×0.75×PR):

| 약화된 공격자 | vs 전사(Def10) | vs 마법사(Def6) | 원래 값 | 감소율 |
|---|---|---|---|---|
| 전사 기본 floor(40×0.75×1.0)=30 | **20** | **24** | 30/34 | −33%/−29% |
| 전사 베기 floor(30×1.3)=39 | **29** | **33** | 42/46 | −31%/−28% |
| 마법사 기본 floor(42×0.75)=31 | **21** | **25** | 32/36 | −34%/−31% |
| 마법사 파이어볼 floor(31.5×1.7)=53 | **43** | **47** | 61/65 | −30%/−28% |
| 마법사 치유 | **+33 불변** | +33 불변 | 33 | **0%** (§6 가드레일) |

- **왜 0.75인가(0.5·0.8 기각)**: ×0.5는 전사 기본이 30→10(−67%) — 랜덤 프록이 게임을 결정하는 크기. ×0.8은 −24~27%로 체감 미달. 0.75 = 최종딜 −30% 내외로 "명확히 느껴지되 판은 안 뒤집는" 구간. 추가로 0.75는 **이진 정확(3/4)이라 float-floor 함정이 없다**(§6-4 — 0.7은 40×0.7=27.999…→floor 27로 손계산 28과 어긋나는 실함정).
- **칩락 안전규칙 재검증**: 약화 반영 강화판 = `max(Def) < min(Atk×0.75×1.0)`. 실측 min = 전사1성 Atk32 → floor(32×0.75)=**24 > max Def 13** ✓. 어떤 약화 조합도 기본공격이 1로 바닥나지 않음(스택 없음 전제, §1).
- **지속 2턴인 이유**: 1턴이면 기대 절감 ~10~13(체감 미달), 3턴이면 ~35+(기절 초과). 2턴 = 프록당 기대 절감 **~20~26 원시딜** ≈ 기절 1프록(~35)보다 약간 약함 → 확률을 기절보다 높게(35% vs 25%) 줘서 프록당 가치 × 확률의 **킷 간 EV를 등가로** 맞춘다(§5-3).
- **D6 지속 소진 기준**: **안A(권장) = 자기 턴 경과로 소진**(행동 종류 무관 — 그 턴에 막기/치유를 해도 1턴 소모). 근거: 틱 규칙 단순(턴 종료 일괄 −1) + **카운터플레이 생성**(약화 걸린 마법사가 그 턴 치유를 돌려 약화 턴을 "흘리기" 가능 = 스킬 선택 유의미화). / 안B = 공격 행동 2회로 소진(행동 종류 검사 필요, 흘리기 불가, 효과 보장은 확실). 
- **틱 타이밍 명세(안A 기준)**: 부여는 상대 턴 중(step8.5) → 피부여자 자기 턴 종료 시 잔여 −1 → 0이면 해제. Duration=2 = **자기 턴 정확히 2회의 행동이 영향권**. 막기(자기 턴 시작 해제)와 틱 시점이 다른 것은 의도 — 막기는 방어 버프(남의 턴에 의미)라 시작 해제가 맞고, ATK_DOWN/STUN은 자기 턴에 의미가 있어 종료 틱이 맞다. 단일화하려고 시작 틱을 쓰면 "Duration=2가 실제 1행동"이 되는 데이터 왜곡이 생긴다(기각 근거).

### 3-3. D7 — 재부여(중복) 규칙
- **안A(권장): 지속 리셋** — 이미 걸린 동종 효과에 재프록 시 잔여 지속 = Duration으로 리셋. 값 중첩·지속 합산 없음. 구현 최단(무조건 대입 1줄), STUN(D=1)에서는 사실상 "무시"와 동일하게 동작.
- 안B: 완전 무시(걸려 있으면 롤 자체 버림) — 더 보수적이나 리셋과 실효 차이가 미미(ATK_DOWN 잔여1→2 정도).

## 4. D-확률 설계 — Chance 대역과 트레이드오프 원칙

### 4-1. 스킬 파워 예산 (기존 5종에서 역산)
쿨다운 티어가 사주는 프리미엄(기본공격 PR1.0 대비, 사용 1회당 원시딜 환산):

| cd 티어 | 기존 스킬 | PR 프리미엄 | 원시딜 환산 |
|---|---|---|---|
| cd0 | 공격 1.0 | 0 (기준) | — · **상태이상 부착 금지 대역** |
| cd1 | 베기 1.3 | +0.3 | **+10~12/사용** |
| cd2 | 파이어볼 1.7 | +0.7 | **+23~29/사용** |
| cd2~3 | 막기·치유 (PR0/0.8) | — | **결정적(100%) 효과 전용 대역** — 행동 자체가 코스트 |

**원칙(제안): 강한 효과일수록 낮은 확률 + 높은 쿨다운.** 부가효과의 기대가치 `EV = Chance × 프록당 가치(원시딜 환산)`가 위 티어 프리미엄과 같은 화폐로 잰다. 이번 부여(§5)는 PR·cd를 안 건드리고 EV를 얹으므로 엄밀히는 "예산 초과"다 — 알파에서 허용하는 이유: (1) **양 킷에 동시 부여해 상대 밸런스 보존**(§5-3 등가), (2) TTK 앵커(기본공격)는 무변경, 스킬의 상대 가치만 상승 = "스킬 쓸 맛" 방향과 일치, (3) 과강 판정 시 **Chance만 내리면 됨** — Chance는 손계산표 셀을 하나도 안 건드리는 유일한 튜닝 노브(1차 노브 = Chance, 2차 = Duration. PR/cd는 최후).

### 4-2. 권장 Chance 값과 대역
| 부여 | Chance | 근거 |
|---|---|---|
| 베기 → STUN | **25%** | 프록당 가치 ~35+템포(강효과 → 저확률). 전사 2기 × 베기 ~2회/판 = 시도 4회 → **팀당 기대 1.0프록/판**. 연속 기절(스턴락) P ≤ 0.25² = 6.3%/시도쌍 |
| 파이어볼 → ATK_DOWN | **35%** | 프록당 가치 ~22(중효과 → 중확률). 마법사 2기 × 파볼 ~1.5회/판 = 시도 3회 → **팀당 기대 1.05프록/판** |
| 가드레일 | **부가효과 대역 20~40%, 50% 초과 금지, cd0 금지** | >50%면 프록이 기본값처럼 굳어 미발동이 벌점으로 반전 + EV 폭증. 100%는 시전형(막기류) 전용 |
| 하향 폴백 | 15% / 25% | 실플레이서 과강 판정 시 1차 조정치(팀당 기대 ~0.6프록/판) |

- **체감 설계**: 팀당 각 ~1프록/판, 양 팀 합 ~4프록/판 = "한 판에 몇 번 터지는 이벤트". 오너가 원한 "확률로 터질 수 있는" 체감을 만족하면서 원장(±1~2턴 변동)을 지배하지 않는 빈도.
- **롤 계약**: 시전마다 **독립 롤 1회**(step8.5), 누적 보정(천장)·명중 롤 없음(명중은 100 고정 유지 — 이 롤이 시스템 유일 난수, §1 개정).

### 4-3. 막기(100% 결정적)와 신규(확률적)가 같은 EffectType 축에 있어도 되는가
**된다 — 단 카탈로그가 위상(phase)을 명시하는 조건으로** (스키마 판단은 D3, §7).
- 개념적으로는 다른 물건이다: 막기 = **시전형**(스킬 그 자체가 효과, 자기 대상, 행동 소비가 코스트, 항상 발동) / 신규 = **명중부가형**(딜 스킬에 얹히는 라이더, 피격자 대상, 확률이 코스트).
- 그러나 TakeHit은 **이미 EffectType 값별로 분기**한다(step3이 DMG_REDUCE를 문자열로 특정). 위상은 이미 코드에 내장된 구조고, 신규는 step8.5 분기 1개 추가일 뿐 — 별도 테이블로 갈라도 이 분기는 사라지지 않는다. 즉 축 분리의 구조적 이득이 없다.
- **계약**: `EffectChance`는 **명중부가형에서만 소비**된다(시전형·Kind형은 롤 미경유, 데이터상 1.0으로 기록해 "항상 발동"을 문서화). 위상 매핑의 SSOT = 본 카탈로그 표(§3).

## 6. 합성 순서 수식 (광폭화·막기·약화 동시)

### 6-1. D4 — 적용 방식: 국소 배율(안A, 권장) vs Atk 스탯 변이(안B)
- **안A(권장): step4 국소 배율.** 광폭화(F1 확정)와 **동일 원칙, 동일 근거**:
  1. **치유 오염 차단** — 치유(step2)는 `Attacker.Atk`를 그대로 읽는다. 스탯을 깎으면 약화된 마법사의 힐이 33→24로 같이 깎임(효과 하나가 딜+힐 두 축을 건드림 = 밸런스 추적 불가). 광폭화가 step4 국소를 택한 바로 그 이유.
  2. **원복 버그 면역** — 스탯 변이는 해제 시 원복 단계가 필요(누락 = 영구 약화 버그). 국소 배율은 상태 플래그가 꺼지면 식이 자동 복귀(원복 코드 자체가 없음).
  3. **F8 TC 보존** — 스탯 변이면 광폭화 기대값표(TC-Z01~05)가 약화 중 전부 어긋난다. 국소 배율이면 두 배율이 같은 자리에서 곱으로 독립 합성.
- 안B: Atk 스탯 직접 감소 — "공격력 약화니까 힐도 약해져야" 관점이면 후보. 기각 근거: 힐-Atk 연동은 알파의 데이터 절약이지 세계관 확정이 아니고, 힐 약화가 필요하면 별도 효과(HEAL_DOWN, 베타)로 명시 도입이 맞다. 가드레일 일관성: **"스탯은 절대 변이하지 않는다, 모든 일시 효과는 판정식의 국소 배율"** 한 줄로 엔지니어 규칙 통일.

### 6-2. 확장된 판정 순서 (v1 §8의 개정 제안 — step4 확장 + step8.5 신설)
```
4.   base = floor(Attacker.Atk × AtkDownMult × PowerRate × BerserkMult(TurnCounter))
       · AtkDownMult = Attacker에게 ATK_DOWN 활성이면 0.75, 아니면 1.0
       · 단일 floor(광폭화와 동일) + 곱 순서 고정: Atk → AtkDownMult → PR → BerserkMult
5.   dmg = base − Target.Def                                  (기존)
6.   Target에 DMG_REDUCE 활성 시 dmg ×= (1 − 0.5)             (기존)
7.   dmg = max(1, floor(dmg))                                 (기존)
8.   HP 적용·사망 판정                                         (기존)
8.5. [신설] 부가효과 롤: Target.bAlive == true
       AND 스킬 EffectType ∈ {STUN, ATK_DOWN}
       AND rand() < EffectChance
       → Target에 효과 부여(잔여 지속 = EffectDurationTurns, 재부여=리셋 D7)
9.   쿨다운 세팅·로그                                          (기존)
```
- **롤은 사망 판정 뒤**(시체에 상태이상 부여 금지 — 킬링 블로우는 롤 자체를 생략).
- step2(HEAL)·step3(DMG_REDUCE)는 **미접촉** — 치유·막기 값은 약화·광폭화와 무관하게 불변(광폭화 가드레일 그대로 확장).
- 막기가 피해를 반감해도 **롤은 진행**(D9-안A 권장: 막기 가치는 반감으로 이미 지불됨. 안B=막기 중 프록 차단은 막기를 "예측 방어" 카운터로 강화하는 베타 후보로 예약).

### 6-3. 체인 검산 (3효과 동시 — 손계산 정답지)
turn 40(BerserkMult=1.5), ATK_DOWN 걸린 마법사가 파이어볼 → 막기 중 전사:
```
base = floor(42 × 0.75 × 1.7 × 1.5) = floor(80.325) = 80
dmg  = 80 − 10 = 70  → 막기 ×0.5 = 35 → max(1, floor(35)) = 35
비교(약화 없음): floor(42×1.7×1.5)=107 − 10 = 97 → ×0.5 = 48
```
평시(turn≤30) 조합: 파볼(약화) vs 막기 전사 = floor(53.55)=53 − 10 = 43 → ×0.5 = **21** (원래 30).

**종결 보장 재검**([[광폭화_재검증]] §5의 부등식에 약화 최악 가정 대입): 공격자 전원 상시 약화라는 비현실 상한에서도 `62×0.75×m − 15 > 66 → m > 1.74 → turn ≈ 45`. 임계가 37→45로 밀릴 뿐 **유한 종결 유지**(광폭화 무상한 + 약화 uptime 현실값 ~20%대에선 ~38 근방). 기절은 수비자가 "공격(베기)해야만" 나오므로 스톨을 돕지 않음. 순수-수동 홀은 기존 결론 그대로(공격 0 = 프록도 0).

### 6-4. float-floor 함정 (0.7 기각의 실측 근거)
IEEE754 double에서 `0.7`은 0.69999999999999996으로 저장 → `40×0.7 = 27.999…` → **floor 27 ≠ 손계산 28** (검증 미스매치 실함정). `0.75`는 이진 정확(3/4) → `40×0.75 = 30.0` 정확. 신규 배율이 낄 전 셀(§3-2 매트릭스·§6-3 체인)과 기존 전 셀(PR 1.3/1.7/0.8, Mult 1.05/1.25/1.5/2.0)을 곱 순서 고정으로 float 검산 완료 — **전부 손계산과 일치**(0.7만 탈락). 향후 배율 추가 시 규칙: **이진 정확값(0.25/0.5/0.75/1.5…) 우선, 아니면 float 검산 필수.** (전 floor에 +1e-6 엡실론을 넣는 공학적 대안도 있으나 기존 F8 기대값 전 셀 재검이 따르므로 이번 범위에선 값 선택으로 회피 — 향후 미스매치 표면화 시 Director 판단.)

- 상태이상 표시용 문자열 키(`Status.Stun` 등)는 **A1 범위 밖**(HUD 부재, 검증은 로그로만) — 표시 도입 시 strings.csv 키 신설 예약(하드코딩 금지 규칙 선제 메모).

## 8. 검증 전략 (결정론 개정의 보상책)

### 8-1. 정답지 계층
| 계층 | Chance 강제 | 유효한 정답지 |
|---|---|---|
| L0 회귀 | **0%** | 기존 §5-1 전 셀 + 23턴 even-trade 원장 **무변경 재사용**(basic 전용이라 원래도 무프록) |
| L1 프록 경로 | **100%** | 본 문서 §3-2 약화 매트릭스·§6-3 체인·기절 스킵 시나리오(결정론 복원) |
| L2 실플레이 | 실확률 | 로그의 프록 기록을 입력으로 한 **사후 결정론 대조**(§8-3) |

### 8-2. D10 — 검증 모드 훅 (요구사항)
밸런스가 요구하는 것은 **"검증 시 Chance를 0/1로 강제할 수 있는 수단"의 존재**뿐이다(전역 오버라이드 변수/스캐폴드 직접 부여/테스트 DT 중 택1 — 수단 선택은 gameplay·Director 소관). 이 훅 없이는 상태이상 TC가 전부 비결정 → verifier 실증 불가.

### 8-3. 로그 가시성 요구 (정확한 스키마는 F0⑧ 오너인 Director/구조 소관)
사후 대조가 성립하려면 로그에 최소 3가지가 남아야 한다: ① 시전별 **프록 여부**(효과 종류 포함) ② **기절 스킵 발생**(해당 턴이 소모됐음) ③ (권장) 잔여 지속. 프록이 로그에 있으면 실확률 게임도 각 라인이 결정론 검증 가능(프록 열을 정답지 분기 입력으로 사용) — F9a 방법론이 확률 도입 후에도 생존하는 유일한 경로.

### 8-4. TC 시드 (qa-critic 인계)
- [TC-SE01] Chance=0 강제 → §5-1 전 셀 + 23턴 원장 무변경(회귀).
- [TC-SE02] Chance=1 베기 → 피격자 다음 자기 턴 1회 스킵, 그 다음 자기 턴 정상 행동(시한부 증명).
- [TC-SE03] Chance=1 파볼 → 피격자 후속 공격이 §3-2 약화 매트릭스와 일치(자기 턴 2회), 3회째 원복.
- [TC-SE04] 약화 중 치유 = **33 불변**(step2 미접촉 — TC-Z03 자매).
- [TC-SE05] 킬링 블로우 → 프록 롤 생략(시체 상태이상 0).
- [TC-SE06] 기절 턴 쿨다운 −1 진행 + 막기 버프는 기존 계약대로 해제.
- [TC-SE07] 기절 턴 TurnCounter +1 (D8 채택 시).
- [TC-SE08] 3효과 체인 → §6-3 값(turn40 파볼·약화·막기 = 35 / 평시 21).
- [TC-SE09] 재부여 = 지속 리셋(중첩·연장 없음, D7).
- [TC-SE10] 기절자 타겟팅·치유 가능(사망자와 구분), bAlive 무오염.
- [TC-SE11] 칩락 안전: 약화 1성 전사 기본(24) − Def13 = 11 ≥ 1 (min1 미발동 확인).
- [TC-SE12] 로그: 프록/스킵 기록 존재(§8-3 ①②).
