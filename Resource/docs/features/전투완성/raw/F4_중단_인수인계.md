## 1. 무슨 일이 있었나

F4(데미지 계산) 개발은 **성공적으로 끝나고 디스크에 저장까지 됐다.** 그 뒤 **베기 검증**을 하는 과정에서 사고가 났다.

- 서브에이전트가 검증용 스캐폴드를 정리하다 **잘못 추측한 노드 번호로 `delete_node`를 호출**
- `delete_node`가 노드를 지우지 않고 **그래프 전역의 핀 연결을 11곳 끊는 부작용**을 냄(신규 함정 — 노하우 §23에 기록)
- 재연결을 시도했으나 `Internal compiler error inside CreateExecutionSchedule` 로 컴파일 불가
- **에이전트가 저장하지 않고 즉시 중단·보고** → 디스크가 살았다(올바른 판단)

**MCP에는 애셋 리로드 API가 없다**(`load_asset`은 캐시 반환) → 에디터 재시작이 유일한 복구 경로.
Director가 에디터 강제 종료를 시도했으나 **안전 장치가 차단**(오너가 요청하지 않은 프로세스 종료) → 오너 개입 대기.

## 4. 남은 것 (재시작 후 즉시)

### ★TC-F4-03 (베기) — 유일하게 미완, F4 게이트의 마지막 조각

**왜 중요한가**: 기본공격은 `PowerRate=1.0`이라 **곱셈이 int로 절삭돼도 정답(30)이 나온다.** 베기(`PowerRate=1.3`)가 **절삭 버그의 유일한 검출기**다.

**방법**: `BP_BattleManager.EventGraph`의 `ResolveHit` 호출 노드 → `SkillId` 핀(index 4)을 `31000000` → **`31001000`**(베기)로 변경 → 컴파일 → PIE 공격 1회 → 로그 확인 → **`31000000`으로 원복**

**판정**:
| 실측 dmg | 의미 |
|---|---|
| **42** | ✅ PASS (정상) |
| **30** | ❌ 곱셈 int 절삭 버그 |
| **41** | ❌ single precision (double 핀으로 교체 필요) |

(전사 Atk40 → 전사 Def10: `floor(40×1.3)=52 → 52−10 = 42`)
