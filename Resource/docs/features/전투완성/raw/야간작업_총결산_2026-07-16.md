## 2. 오너 육안 게이트 목록 (엔진에서 직접 볼 것)

이 항목들은 기계 검증(데이터소스 정합)까지만 끝났고, 룩·사용감은 오너 육안이 최종 판단이다.

1. **데미지폰트 룩** — 위치·페이싱(2.4s)·다중 겹침 가독성 (③·③b)
2. **유닛 추적** — 카메라 컷(액션캠) 중 데미지 숫자가 자연스러운지 (③b 이월)
3. **스킬메뉴 사용감** — 전사(공격/베기/막기), 마법사(파이어볼/치유) 4종 연출 (F7a)
4. **막기 반감 체감** — 데미지 절반 감소가 체감상 자연스러운지 (F7a)
5. **End→재전투 루프** — 전환 타이밍·페이싱 (④)
6. **풀플레이 1판** — 광폭화 포함 30턴+ 시나리오 일괄 확인(F9b 몫, F8은 단독 확인 생략하고 여기 포함하기로 기확정)

## 3. 오너 결정 대기 2건

### ① F7b struct 생성 방식

MCP는 UE struct 자체를 생성하지 못한다(엔진 API 제약) — F7b(3층 데이터 인터프리터)에 필요한 `F_SkillEffectRow`·`F_StatusEffectRow` 2종 + 기존 `F_SkillsRow` 컬럼 추가가 오너 개입 대상이다([[F7_TC]] B01 스펙).

| 안 | 방법 | 소요 | 비고 |
|---|---|---|---|
| (a) | 오너 수동 생성 | ~10분 | F2 때 이미 해본 방식(선례) — struct 2종 + `F_SkillsRow` 컬럼 추가만 하면 됨, 추가 설치 불요 |
| (b) | TAPython 플러그인 승인 | 설치 + 에디터 재시작 | 무료·UE5.8 지원. **AGPL-3.0 라이선스 검토 필요**. 승인되면 이후 struct 작업 전부 스크립트화(F7b 이후 단계에도 재사용 가능) |

두 안 다 실행 가능 — 오너 선택 대기.

### ② push 승인

커밋 다수(③~F8, F9a는 커밋 예정) push 대기. 단독 push는 규칙상 금지 — 오너 확인 후 진행.

## 4. 발견·수정 이슈 요약

전부 근본원인을 실측한 뒤 수정했고, [[언리얼_MCP_실전노하우]] §28~§32 (54)~(71)에 적재했다.

- **가짜 GREEN 2건**(F6) — FC≥1 가드 분기 반전 + `SetPendingSkillId` 순서결함. 둘 다 "결과값이 우연히 정답과 일치"해 겉보기엔 정상으로 보였던 케이스.
- **`FunctionEntry` 미배선 시스템 함정**(3재발) — 컴파일 0에러가 "본체 실행됨"을 보장하지 않는 구조적 함정, 한 야간 세션 안에서만 3회 재발.
- **`SetInputMode` 부재**(UMG 최초) — 프로젝트 최초의 진짜 UMG 버튼이 클릭에 전혀 반응하지 않던 원인.
- **SELF 거리퇴화 잠복 단선**(F4 유래) — F4 단계부터 미연결이던 카메라 분기가, SELF 대상 스킬이 처음 그 분기를 밟으며 소프트락으로 발현.
- **데미지폰트 카메라 재투영**(③b) — 액션캠 좌표 1회고정·CamBack 후 미재투영이 근본원인.
