## 1. 무엇이 됐나 — 사망·승패 판정

**이전 증상**: 한 팀을 전부 죽여도 전투가 끝나지 않고 무한 반복(소프트락)이었다.

**F5-1 해법**: `EnterTurnEnd`에 `Branch(bBattleOver)` 단일 관문을 추가해 해결.
- `bBattleOver`는 **`ResolveHit`의 사망 분기에서만** 세팅 — "같은 팀 생존수==0"일 때만 true ([[F5_착수지시서]] B1이 정정한 "같은 팀" 카운트 기준을 그대로 따름 — 레이스 방지).
- `True` → `EnterEnd()`(입력잠금 + `BattleFinished` 디스패처 방출)
- `False` → 기존 `Delay(0.35)` → 다음 턴

## 2. 실증 (PIE 로그 대조)

이번 세션 아군 4명이 전부 사망했다.

| 유닛 | 사망 턴 |
|---|---|
| A1 | turn14 |
| A3 | turn16 |
| A2 | turn22 |
| A4 | turn24 |

turn24에서 마지막 아군 A4가 hp=0이 되는 순간, State 로그가 `turn24 TurnEnd`에서 완전히 끊긴다 — turn25가 없다는 것은 `EnterEnd`가 발동해 전투가 정상 종료(입력잠금)됐다는 뜻이다. 적 B4가 8HP로 생존해 **적팀 승리**. 오너 육안으로도 "한쪽편 hp 모두 0 → 멈춤"을 확인했다.

**부수 확인 — 시체를 때려도 승패가 안 깨짐**: death 분기가 "살아있는 같은 팀 수"로 카운트하므로 시체 재타격으로는 조기종료되지 않는다. 예: 이미 죽은 B1을 turn13에 또 때렸으나 다른 적이 생존해 있어 `bBattleOver=false`가 유지됐다.

## 3. F5-1 이월 — F5-2에서 처리할 갭 (버그 아님, 예상된 미구현)

이번 판 실플레이에서 관찰됐다:

- **죽은 유닛이 계속 턴을 받아 공격함** — 죽은 A3가 turn21에 B2를 처치, 죽은 B3가 turn22에 A2를 처치. TS1~TS6 턴스킵이 아직 미구현.
- **시체 클릭/타겟 가능** — 하이라이트는 이미 사망자를 제외해서 정상이지만, `ClickBox`를 NoCollision으로 바꾸는 부분만 미구현.
- **쓰러짐 연출 미구현** — DYING 모션 + `bFreeze` 고정이 아직 배선되지 않음.

## 4. 신규 함정 — "검증 잔여물 + 반쪽 구현 = 라운드2 결정론적 실패"

증상: "전투가 일정 이상 지나면 데미지가 안 들어감"(매 PIE 세션이 정확히 turn=8에서 결정론적으로 정지).

**복합 원인 2건(둘이 겹쳐야 발현)**:
1. F4 베기 검증 때 `ResolveHit` 호출 노드의 `SkillId` 핀을 `31000000`(기본공격)→`31001000`(베기)로 바꿔 검증했는데, **검증 후 `31000000`으로 원복하는 절차가 아침 복구 작업 중 실행 누락**됐다. 라이브 공격이 베기(쿨다운 1턴)로 고정된 상태였다.
2. `ResolveHit` 진입 첫 노드가 **쿨다운 가드**(`GetSkillCooldown > 0`이면 데미지 계산 전 차단)인데, **쿨다운 감소 로직(F5의 TS3)은 아직 미구현** — 가드는 있고 감소는 없는 반쪽 구현이었다.

**결과**: 라운드1에 8유닛 전원이 베기 사용 → 쿨다운=1 세팅 → 안 풀림 → 라운드2(정확히 turn=8, 8유닛 한 바퀴 직후)에 전원이 가드에 막혀 데미지·로그 없이 걸음만 반복. 매 PIE 세션이 결정론적으로 turn8에서 멈췄다.

**진단법**: "일정 이상 지나면"이라는 모호한 증상을 BattleLog와 대조하니 매번 정확히 turn=8에서 정지하는 **결정론적 패턴**을 발견 → 유닛 수(8)와 일치 → 라운드 경계를 의심 → BP 그래프를 `find_nodes`/`get_node_infos`로 직접 추적(PIE 중엔 `read_graph_dsl`이 빈값을 반환하므로 노드 단위 조회가 필요)해 `ResolveHit` step0의 쿨다운 가드를 근본원인으로 특정했다.

**수정**: `SkillId` 핀 `31001000`→`31000000` 원복(계획상 F4~F5 단계의 라이브 공격은 기본공격이 정상, 베기 등 스킬은 F7 스킬메뉴+쿨다운감소와 함께 결선). 컴파일 0에러 + 저장 완료. 기본공격은 쿨다운 0이라 가드에 안 걸린다.

**교훈**:
- (a) 검증용으로 바꾼 디버그 값은 검증 직후 즉시 원복하고, 원복 단계를 체크리스트화할 것.
- (b) "가드는 있는데 소비/감소는 미구현"인 반쪽 기능은 결정론적 지뢰다 — 반쪽만 넣지 말거나, 넣었으면 그 기능을 트리거하는 데이터(쿨다운>0 스킬)를 라이브 경로에서 격리할 것.
