## 2. 공식 선택 — 감산형 채택

Director 후보 2종을 동일 조건으로 검토:

| 후보 | 식 | 장점 | 알파 리스크 |
|---|---|---|---|
| **감산형(채택)** | `Dmg = Atk×PowerRate − Def` (min 1) | 직관·손계산 쉬움, HD-2D JRPG 관례 | Def가 Atk×PR 이상이면 1로 바닥(칩뎀 교착) |
| 비율형 | `Dmg = Atk×PR × Atk/(Atk+Def)` | Def 체감 감쇠, 교착 없음 | 곱셈·나눗셈, Atk 이중 등장(덜 "간단") |

**채택 = 감산형.** "간단이 지상명령"(Director)에 부합. 비율형의 유일한 우위(교착 방지)는 **설계 규칙으로 회피** 가능:

> **칩뎀-락 회피 규칙(불변): 로스터 전체에서 `max(Def) < min(Atk × 1.0)`.**
> 알파 실측: max Def = 13(전사3성), min 기본공격 위력 = Atk 32(전사1성) → 32 > 13. **어떤 조합도 기본공격이 1로 바닥나지 않음** → 감산형이 안전. 유닛 추가 시 이 부등식을 게이트로 강제.

> 설계 함의: 감산형은 Def 편차가 크면 붕괴한다. 따라서 **탱킹은 Def가 아니라 HP로 표현**하고 Def 편차는 좁게 유지(전사10 vs 마법사6 = 4 차이). 아키타입 차별화는 스킬 킷이 담당.

### 데미지 공식 v1 (확정)
```
기본데미지  = floor(Atk_공격자 × PowerRate)
피해        = 기본데미지 − Def_대상
막기 활성 시 피해 ×= (1 − 0.5)              # DMG_REDUCE 버프
최종피해    = max(1, floor(피해))            # 최소 1 보장 (회복/버프는 이 경로 미경유)
회복(HEAL)  = floor(Atk_시전자 × PowerRate)  # 부호 반전, Def·min1 미적용, MaxHp 클램프
```

## 6. 엣지 케이스 처리

| 케이스 | 처리 |
|---|---|
| 피해 ≤ 0 (Def ≥ 기본데미지) | `max(1, …)` → **최소 1**. 무한 무피해/교착 방지 |
| 알파 로스터에서 위 케이스 발생? | **없음**(min Atk32 > max Def13). 안전규칙(§2) 유지 시 유지 |
| 막기로 피해 0 근접 | ×0.5 후에도 `max(1,…)` → 최소 1 |
| 회복 > MaxHp | **MaxHp 클램프**(초과분 버림). 회복은 Def·min1 미적용 |
| 힐 대상이 사망 유닛 | 타겟팅 단계에서 **생존 아군만** 후보 → 무효화 |
| 오버킬(HP 큰 음수) | **Hp=0 클램프** 후 사망 처리(음수 미표시) |
| 소수점 | `floor`로 정수화(기본데미지·최종피해·회복 각각) |
| Def 극단값(미래 유닛) | 안전규칙 위반 시 칩락 → **유닛 추가 시 `max(Def)<min(Atk)` 게이트 강제** |

## 7. 사망 · 승패 판정 규칙

1. **사망**: 피해 적용 직후 `Hp ≤ 0` → 즉시 사망(같은 프레임). Hp는 0으로 클램프.
2. **턴큐 스킵(splice 절대 금지)**: 사망 유닛은 큐에서 제거하지 않는다. 큐 length는 전투 내내 불변. 죽은 유닛은 `bAlive=false` 플래그를 세우고, 자기 턴이 도래하면 행동 없이 즉시 스킵(다음 인덱스로). 기존 턴제MVP의 `CurrentIndex = (i+1) % 큐길이` 순환을 그대로 유지 — splice하면 인덱스가 밀려 턴 스킵/중복·전멸 시 %0 크래시가 발생하므로 금지.
3. **승패**: 한 파티의 **전 유닛 사망 = 그 파티 패배**, 상대 승리.
   - **판정 타이밍**: 매 피해 적용으로 사망이 발생한 **직후** 양 파티의 생존 유닛 수를 센다. 한쪽이 0이면 즉시 전투 종료(턴 루프 탈출) — `% 큐길이` 순환이 전멸한 파티를 다시 도는 일이 없도록 사망 시점에 종료.
4. **동시 전멸**: 알파는 **단일 타겟만** → 한 번의 피해가 양측을 죽일 수 없어 **발생 불가**. (반사/도트 도입 시: 피해를 "가한 쪽" 생존 우선 판정 — 베타 예약)
5. **선공 편향(플래그)**: ~~고정 큐라 큐 앞 유닛/먼저 행동하는 파티가 이점. Spd 미사용이라 **알파엔 카운터 없음**. vs-AI 알파에선 허용, **베타 ATB·선공 룰에서 해소 예약**.~~ **→ [2026-07-18 갱신] 스테일.** 파트2 라이브화로 **Spd가 턴순서를 결정**(1회 정렬 후 순환) — "Spd 死컬럼·고정 큐" 전제 붕괴. 오너 S3 결재(2026-07-18)로 Spd 정식 승격. 선공 편향의 카운터 = ①포지션 Spd 밴드가 방어스탯과 교환(딜 빠름·물몸/탱 느림·맷집, [[balance1_포지션킷_v1]] §3) ②Spd는 등급 미스케일(선공 우위와 등급 우위 이중가산 차단) ③드래프트 밴/픽(일급 카운터). **선공 버스트 지배 시뮬 검증은 balance#5로 이월(오너 결정).**
6. **쿨다운 진행(신규)**: 각 유닛의 **자기 턴 시작 시** 그 유닛이 보유한 모든 스킬 쿨다운을 −1(하한 0)한다. 쿨다운이 남은 스킬은 그 턴에 시전 불가(§8 전제).

## 8. TakeHit(Damage) 인터페이스 요구사항 (gameplay-engineer 인계)

> 구현은 gameplay-engineer 소관. 아래는 **필요 파라미터 + 결정론적 판정 순서**(밸런스가 요구하는 계약).

**입력 파라미터**
| 파라미터 | 용도 |
|---|---|
| `Attacker` (유닛 ref) | Atk 읽기, 로그·연출 |
| `Target` (유닛 ref) | Def·Hp·버프 상태 |
| `SkillId` (또는 해석된 PowerRate/Kind/EffectType/EffectValue/Duration) | 스킬 파라미터 로드 |

**판정 순서(고정)**
```
0. 시전 전제: 시전 유닛의 SkillId 쿨다운이 0인지 확인 — >0이면 시전 불가(AI/입력 단계에서 후보 제외). MP 없음(알파).
1. 스킬 해석: PowerRate, Kind, EffectType, EffectValue, EffectDurationTurns 로드
2. Kind == HEAL      → Target.Hp = min(MaxHp, Hp + floor(Atk × PowerRate)); goto 9
3. EffectType == DMG_REDUCE(막기) → Target(=self)에 감소버프(Value, Duration) 부여; goto 9  (피해 없음)
4. base = floor(Attacker.Atk × PowerRate)
5. dmg  = base − Target.Def
6. Target에 활성 DMG_REDUCE 버프 있으면 dmg = dmg × (1 − BuffValue)   # 방어감산 후 곱
7. dmg  = max(1, floor(dmg))                                          # 최소 1 (마지막)
8. Target.Hp -= dmg;  if Hp ≤ 0 → Hp=0, bAlive=false, 승패체크(§7-3)   # 큐 splice 없음
9. 시전 성공 처리: 시전 SkillId의 쿨다운을 CooldownTurns로 세팅; return { AppliedDamage, bDied }
```
**순서 근거**: 방어(감산)를 먼저 → 막기(받는 피해 감소)를 그 뒤 곱 → 최소1은 맨 마지막(막기로 0이 돼도 1 보장). 회복·버프는 데미지 경로(5~7) 미경유. 전 과정 난수 없음(결정론 = TTK 검증 유효).

**버프 처리 요구(확정)**: DMG_REDUCE(막기)는 **횟수 무제한 지속형** — 시전자의 다음 자기 턴이 올 때까지 받는 모든 피해에 ×(1−Value)를 적용한다(피격 1회로 소비되지 않음). **해제 시점**: 시전자의 자기 턴 시작 시(§7-6 쿨다운 감소와 같은 타이밍)에 버프 제거. EffectDurationTurns=1은 "다음 자기 턴까지"를 의미. 막기는 그 턴의 행동을 소비하므로(Target=SELF, 공격 안 함) 전원 막기 시 딜 0 = 승리 불가 → 터틀 지배전략 성립하지 않음. 쿨다운은 유닛별 (SkillId → 남은 쿨턴) 상태로 저장(§7-6에서 감소).

### 10-2. top-down 참조 위반 없음
```
job_stats(개체) ──JobId──▶ jobs(어휘) ✔ 값 사용
```
