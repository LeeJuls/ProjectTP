# 화면 상단 턴 카운터 표시 — 완료

> 대상: 오너 요청("화면 상단에 턴 표시. 캐릭터 한번 행동할 때마다 1턴씩 증가") · Director 지시서(2026-07-17/18) · 1차 중단(PIE 활성) → Director의 PIE 종료·백업 완료 확인 후 재개.
> 소관: umg-engineer. 카운터 로직(`BP_BattleManager.TurnCounter`)은 **무접촉**(게터만 읽음).

## 1. 무엇이 됐나

`WBP_BattleHUD` 상단 중앙에 `Txt_TurnCount`("턴 {N}") 표시를 추가했다. 값은 `BP_BattleManager.TurnCounter`를 매 프레임 읽어 갱신한다. `BP_BattleManager`는 편집하지 않았다(게터 노드만 신설 그래프에서 호출).

## 2. 설계 결정

- **참조 획득 경로**: `WBP_BattleHUD`엔 기존에 매니저 참조가 없었음(변수: `ActiveUnit` 1개뿐). `WBP_SkillMenu.SetActiveUnit`이 이미 쓰던 **lazy-fetch 패턴**(`IsValid(ManagerRef)` → Not Valid 시 `GetAllActorsOfClass(BP_BattleManager_C)` → `Array Get[0]` → `Set ManagerRef`)을 그대로 복제해 `WBP_BattleHUD`에 신설. `TurnCounter`는 `find_node_types`로 외부 접근 가능(Private 아님) 확인 후 게터만 사용.
- **함수 바인딩 방식**: UMG 네이티브 "Bind" 드롭다운(`UWidgetBlueprint::Bindings`)을 설정하는 MCP 도구를 찾지 못해(UmgMcp·BlueprintTools·ObjectTools·UMGToolSet 전수 조사, `BindToEventProperty`는 멀티캐스트 델리게이트 전용으로 확인) **Director 승인 하에 Tick 방식으로 대체**: `WBP_BattleHUD`에 이미 존재하던 미사용 `Event Tick`(K2Node_Event_2)에 `GetTurnCounterText()` 호출 → `Txt_TurnCount.SetText` 배선. 오너 조작 불요, 자동 갱신.
- **문자열 키**: 신규 키 `Battle.TurnCount` = `"턴 {0}"` (기존 `Battle.UnitLabel`="{0} | {1}" 포맷 문자열 선례와 동일 패턴). `MakeTextFromStringTable(Advanced)`로 포맷 문자열 자체를 ST_UI에서 가져오고, `FormatText`로 `TurnCounter`(정수)를 `{0}`에 대입 — 포맷 문자열도 숫자도 하드코딩 없음.
- **배치·스타일**: 화면 상단 중앙(Anchor/Alignment 0.5,0 · Position 0,14), 배경 `Border_TurnCounter`(`--tp-panel-bg` 계열 톤 84% 알파) + `Txt_TurnCount`(Bold 13px, gold `#E8C384`). **placeholder 명시**: 프로젝트에 Cinzel 폰트 애셋이 아직 임포트되지 않아(오너_UMG배치가이드 §0-2 기지정 사항) 기본 폰트(Roboto Bold)로 배치 — spec의 폰트 지정과 다른 지점, 폰트 임포트 후 일괄 교체 가능. 정확한 위치/크기는 오너 육안 미세조정 대상.

## 3. 그래프 구조 (핵심 노드 refPath)

### 3-1. `WBP_BattleHUD:GetTurnCounterText` (신규 함수, Text 리턴)

```
K2Node_FunctionEntry_0.then
  → K2Node_MacroInstance_0 (Utilities|IsValid, InputObject=K2Node_VariableGet_0.ManagerRef)
      Is Not Valid → K2Node_CallFunction_0 (액터|GetAllActorsOfClass, ActorClass=BP_BattleManager_C)
                        .then → K2Node_VariableSet_0 (SetManagerRef, value=K2Node_GetArrayItem_0.Output)
                        .OutActors → K2Node_GetArrayItem_0 (Utilities|Array|Get(사본), Dimension1=0)
      Is Valid ─┐
      VariableSet_0.then ─┴→ K2Node_FunctionResult_0.execute  (두 경로 동일 실행핀에 병합, 실측 확인)

K2Node_VariableGet_1.ManagerRef → K2Node_VariableGet_2.self (Class|BPBattleManager|GetTurnCounter)
K2Node_VariableGet_2.TurnCounter → K2Node_FormatText_0."0"  (add_node_pin으로 별도 생성, 인티저로 자동승격)
K2Node_CallFunction_1 (MakeTextfromStringTable(Advanced), TableId=/Game/UI/ST_UI.ST_UI, Key=Battle.TurnCount)
  .ReturnValue → K2Node_FormatText_0.Format
K2Node_FormatText_0.Result → K2Node_FunctionResult_0.ReturnValue
```

⚠ **함정 실측**: `FormatText` 노드의 `Format` 핀에 리터럴 텍스트를 먼저 세팅해 인자 핀("0")을 자동생성시킨 뒤, `Format` 핀을 동적 소스(MakeTextFromStringTable 출력)로 재연결하면 **자동생성됐던 인자 핀이 사라진다**(엔진이 리터럴 텍스트 파싱 기반으로 인자 핀을 동기화하는데, 핀이 연결되면 파싱 대상이 사라지는 것으로 추정). 해법: `add_node_pin(node)`로 인자 핀을 **명시적으로 재생성**(네이티브 AddPinInterface 직접 호출, 리터럴 텍스트 파싱에 의존하지 않음) 후 연결. 노하우 문서 등재 권장 대상(§34 이후 번호).

### 3-2. `WBP_BattleHUD:EventGraph` (기존 Tick 활용)

```
K2Node_Event_2 (기존 Tick, 이전엔 미사용).then
  → K2Node_CallFunction_0 (|GetTurnCounterText 호출)
      .then → K2Node_CallFunction_1 (위젯|SetText(Text), self=K2Node_VariableGet_0.Txt_TurnCount)
      .ReturnValue → K2Node_CallFunction_1.InText
```

전부 `get_node_infos`로 실핀 연결 재확인 완료(§34(80)(86) 준수 — `find_nodes`/`find_node_types`로 얻은 refPath·type_id만 사용, 직접 조합 안 함).

## 6. 1차 중단·재개 경위 (참고)

착수 세션에서 `IsPIERunning()=true`를 뒤늦게 발견 — ST_UI StringTable 저장이 PIE 중 불가능한 것(§34(81))과, `AssetTools`의 `exists`/`get_asset_class`/`duplicate`/`create_folder`/`save_assets`/`is_dirty`가 광범위하게 오작동하는 것을 동시에 관측해 중단·보고했다. Director가 `IsPIERunning()=false` 직접 실측 + Bash로 3종 백업(SHA256 검증)을 완료한 뒤 재개 지시, 재개 시 `AssetTools` 전 기능이 정상 동작함을 재확인해 **PIE가 근본 원인이었음이 사실상 확정**됐다(별도 노하우 등재는 Director 판단 대상).
