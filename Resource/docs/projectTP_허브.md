## 🔧 환경 메모

- 엔진: `D:\unreal\UE_5.8` (Paper2D ✅ / GameplayAbilities ✅ / Unreal MCP·Toolset Registry 내장)
- MCP: UE 5.8 공식 Experimental. HTTP `http://127.0.0.1:8000/mcp` (loopback, 무인증). 에디터 안에서 기동. **+ UmgMcp**(서드파티, UE5.8 전용 UMG 특화·MIT, 2026-07-14 추가) — stdio(`uv run`), 별도 discovery 포트라 공식 MCP와 충돌 없음. 상세 [[언리얼_MCP_실전노하우]] §21.
- 에셋: heroes99 (au-pixel) — $18.75, 상업·수정 가능, 32px 도트 정면. 무료 데모 없음.
- 배경: **FANTASTIC Village Pack** (Fab 무료) — `/Game/Fantastic_Village_Pack/`, 완성맵 3종(day/night/overview) 포함. UE Content는 git 제외(재현 방식).
- 병행: 오너가 별도 Unity 프로젝트 작업 중 — 충돌 없음(별 엔진/포트). RAM만 주의.

## ✅ 결정 로그

- 장르 = A형(정면 대치 턴제). 필드 없음 → heroes99 방향 스프라이트 부재 문제 해소.
- 최종형 = 실시간 ATB **PvP**(사람 vs 사람). 단, **아트 선검증(vs AI, 클라 only)을 먼저** — 이 게이트는 **완료**(GO 판정). ※그래픽 검증 게이트였음 — "알파에서 게임 시스템(드래프트 등)을 미룬다"는 뜻이 아님(방향성2 재작성 시 명확화).
- 서버 권위·네트워킹은 후순위(사람이 풀 수 있는 영역으로 판단) → **아키텍처 확정(2026-07-12)**: Steam ID+EOS 매칭+리슨서버 P2P+Player Data Storage(서버비 ~0). 상세: [[로드맵_버전계획]] 네트워킹 섹션.
- 프로젝트명 = `projectTP` (※ 임시명 여부 미확정 — 확정 시 폴더/.uproject 동시 변경 필요).
- 문서 관리 = Obsidian 볼트 `Resource\docs`. 엔진 폴더엔 문서 금지.
- 팀 = **Director(메인) + 9 전문 에이전트**. 개발 = **단계별 게이트**(qa=TC설계, verifier=실행, Director=판정).
- **Fable=두뇌**: 작업 100 중 앞10(계획·방향)+뒤10(방향검수)만 Fable, 중간80(실행)은 Sonnet/Haiku. 노동에 Fable 금지.
- MVP = Blueprint-only. C++(cpp-engineer)는 MVP 이후 코어 이관 시 활성화(VS2022 필요).
- 배경 = Fab 무료 Village Pack 채택. UE Content·유료원본·프로젝트 사본은 git 제외.
- **버전 로드맵(2026-07-08 최초 → 2026-07-13 방향성2 재작성)** = 알파(클라 단독: **드래프트(Roll6→Pick5→Ban1 교차밴)→광폭화 포함 4:4 턴제 vs AI→결과**, 8화면 루프)→베타(서버+ATB 전환+유저 주도 조립+플랫폼 확정)→라이브(배포). 상세: [[로드맵_버전계획]].
- **등급(GradeId) 축 = 유지+배율압축(2026-07-13 오너 재확인)** — 구 "등급 도입 금지 권고"(07-10)는 철회. 등급은 작은 신호/양념(밴 휴리스틱·기대감·역전서사), 압축 배율로 순수 RNG화만 억제. 수치는 balance 후속. 상세: [[기획_방향성]] 권고②.
- **드래프트 3결정(오너 확정 2026-07-13)** = 지급 방식 **B 풀랜덤**(포지션 보장 없음 — 시뮬 하한검증 balance 예약) / 광폭화 **30턴 초과부터 턴당 공격력 +5% 가산**(유닛턴 기준) / 포지션 어휘 **탱·딜·서포터**(구 '힐러' 개명 — 힐 포함 변수 창출 직업). 상세: [[기획_방향성]].
- **모션↔스킬↔무기 연결 = B 느슨결합(오너 확정 2026-07-13)** — 스킬이 모션 소유(skills.csv MotionRow), 무기는 외형 전용(1000풀 생성 시 포지션 성향에 느슨 매칭, 예외 15~20%), 마법 룩은 완드 4색 변형. RowIndex 배선+DT_Motions는 A1 이관. 상세: [[모션연결_규칙안]](E0 실태조사 [[파츠모션_실태조사]] 기반).
- **UmgMcp 도입(2026-07-14)** = 서드파티 C++ 플러그인(winyunq/UnrealMotionGraphicsMCP, MIT)으로 UMG 위젯 배치를 MCP 자동화([[UI_화면_규약]] §B "오너 수동" 정정). 도입 중 이 PC의 `.NET Framework SDK` 미설치로 C++ 빌드 자체가 전면 막혀 있던 선재 결함을 발견·해소(`winget install Microsoft.DotNet.Framework.DeveloperPack_4`, 관리자 권한 필요) — 이 PC가 처음으로 C++ 빌드 가능 상태가 됨(어떤 C++ 플러그인이든 해당, UmgMcp 고유 문제 아니었음). 상세: [[언리얼_MCP_실전노하우]] §21.
