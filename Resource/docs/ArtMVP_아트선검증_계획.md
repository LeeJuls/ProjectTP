# 아트 선검증 MVP — HD-2D 룩 확인 (클라 only, vs AI)

> 작성일: 2026-06-28 / 엔진: UE 5.8 / 상태: **계획만, 구현 미착수**

## 0. 이 단계의 단 하나의 질문

**"3D 무대 위에 32px 스프라이트를 라이팅·포스트프로세스와 함께 놓았을 때, 도트가 뭉개지지 않고 옥토패스급 '디오라마 느낌'이 나는가?"**

여기서 Yes가 나와야 PvP·서버권위 같은 무거운 작업에 투자할 가치가 있다. No면 아트 방향(에셋 해상도/스타일)을 다시 잡아야 한다. → **가장 싸게, 가장 먼저 검증.**

## 1. 범위 (의도적으로 최소)

| 포함 | 제외 (이번엔 안 함) |
|---|---|
| 클라이언트 only, 싱글 프로세스 | 네트워킹·서버 권위·PvP |
| 내 캐릭터 1체 + 적 1체(AI) | GAS, 복잡한 스킬 트리, 상태이상 |
| 가장 단순한 턴제: 공격→상대공격 반복 | ATB 실시간 게이지(턴 고정으로 단순화) |
| HD-2D 무대·라이팅·포스트프로세스 | 매치메이킹, 세션, 저장 |
| idle 루프 + 공격 1종 애니(플립북) | PaperZD(유료) — Paper2D 플립북으로 대체 |

> AI는 "정해진 타이밍에 기본 공격"하는 더미면 충분. 이번 목표는 _전투 재미_가 아니라 _화면 룩_이다.

## 2. 성공 기준 (측정 가능하게)

오너가 에디터/플레이 화면에서 다음을 눈으로 확인:
1. **도트 보존**: 캐릭터 스프라이트의 픽셀이 또렷(블러/뭉개짐 없음, 정수배 스케일).
2. **공간감**: 캐릭터가 3D 무대 위에 "서 있는" 느낌(접지 그림자/무대 깊이).
3. **빛 통합**: 무대 라이팅·블룸이 캐릭터를 씬의 일부처럼 보이게 함(스티커처럼 떠 보이지 않음).
4. **분위기**: 약한 DoF·컬러그레이딩으로 미니어처 디오라마 톤.
5. **종합 판정**: "이 방향으로 가도 되겠다"는 오너의 Go/No-Go.

→ 1~4를 만족하는 **정지 스크린샷 1~2장**이 이 MVP의 최종 산출물.

## 3. 룩 타겟 — HD-2D 렌더링 레시피 (구현 시 이대로)

### 3.1 카메라
- Perspective(원근), 살짝 내려다보는 각(피치 -15~-25°), FOV 좁게(망원 느낌 → 미니어처감).
- 캐릭터는 빌보드(카메라 Yaw 정렬) 또는 고정 평면 — MVP는 **고정 평면**으로 시작(정면 대치라 충분).

### 3.2 무대 (깊이의 원천)
- 단순 3D: 바닥 플레인 + 뒤쪽 기둥/벽 몇 개 + 포인트라이트 소품(횃불 등).
- 무대가 "HD"를 담당, 캐릭터는 "2D"를 담당 → 둘의 대비가 HD-2D의 핵심.

### 3.3 스프라이트 머티리얼
- **Lit** 머티리얼(Unlit 아님) — 무대 빛을 받게.
- 라이팅 기여도는 머티리얼에서 조절(과하면 도트 톤 죽음).
- 노멀맵 없음 → MVP는 평면 라이팅 + 환경광으로 "분위기"만. (노멀맵 생성은 폴리시 단계로 보류)
- 마스크드/반투명 처리로 외곽 알파.

### 3.4 픽셀 퍼펙트 (도트 보존의 핵심)
- 텍스처: **Filter=Nearest**, Mip 비활성, 압축은 도트 보존 설정(UI/2D용).
- 스프라이트 스케일은 **정수배**로(0.5배 같은 비정수 금지 → 픽셀 깨짐).

### 3.5 포스트프로세스 (Post Process Volume)
- Bloom: **약하게** — 무대 발광 소품 강조용. 캐릭터까지 번지면 도트 뭉개짐.
- Depth of Field: 전/후경에만, 캐릭터 평면은 또렷.
- Tonemapper + 약한 Vignette + Color Grading(따뜻한 톤 LUT 권장).
- Lumen/Nanite: 32px 도트엔 과함 → **MVP는 끔.** 무대 라이팅은 정적/간단 라이트로.

### 3.6 접지 처리
- 캐릭터 발밑 **가짜 그림자**(블롭 데칼 또는 반투명 타원) → "떠 있음" 방지. 룩에 결정적.

1. 원본 위치(권장): `D:\unreal\Resource\_RawAssets\heroes99\` 에 풀어두기(프로젝트 Content 밖, 소스 보관).
2. 임포트 시 텍스처 설정 강제: Nearest / no-mip / 도트 압축.
3. 시트 → 스프라이트 슬라이스 → **플립북**: 최소 `Idle`(루프), `Attack`(1회). 필요 프레임만.
4. Content 배치: `Content/Sprites/Heroes99/`, `Content/Flipbooks/`.

## 5. 구현 단계 (승인 후 이 순서로)

> 각 단계 "구현→오너 확인" 후 다음. 검증 없이 진행 금지.

- **S0. 파이프라인 더미 검증(선택, 강력 권장)**
  32px 임시 스프라이트로 §3 레시피 씬을 먼저 구성 → 룩 검증. 에셋 의존 없이 엔진 리스크 제거. 통과 시 텍스처만 스왑.
- **S1. 프로젝트 생성**: UE5.8 클라 전용, Paper2D 활성. (BP 기반이 아트 MVP에 빠름; C++ 불필요)
- **S2. 임포트 자동화**: Python으로 heroes99 → 텍스처(Nearest)→스프라이트→플립북.
- **S3. 무대+라이팅+카메라**: §3.1~3.2 레벨 구성.
- **S4. 스프라이트 머티리얼(Lit) + 픽셀퍼펙트 + 접지 그림자**: §3.3~3.6.
- **S5. 포스트프로세스 볼륨 튜닝**: §3.5.
- **S6. 더미 턴제 루프**: 내 캐릭터 idle→Attack 재생, AI 캐릭터 반격. 화면 완성.
- **S7. 스크린샷 추출 + 오너 Go/No-Go.**
